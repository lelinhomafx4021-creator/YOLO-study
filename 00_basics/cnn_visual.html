📥 第0步：输入图像

你喂给模型的是一批 CIFAR-10 图片。假设 batch_size=64：

X 形状 = [64, 3, 32, 32]

16 个核 = 16 张输出特征图

每个核学会了检测一种模式。核1可能检测竖直边缘→遇到边缘输出大正值。核5可能检测红→蓝过渡。 这些不是人指定的，是训练学出来的。你只说"给我16个核"，数据决定它们各自学会检测什么。

输出形状 = [64, 16, 32, 32] padding=1 保持尺寸不变：32×32 → 32×32
⬇

⚡ 第2步：ReLU() — 负数归零

逐个元素操作：ReLU(z) = max(0, z)

[0.8, -0.3, 0.5, -1.2, 0.1] ──ReLU──▶ [0.8, 0, 0.5, 0, 0.1]

只保留"激活"的信号，负的扔掉。形状不变。

📐 第3步：MaxPool2d(2) — 压缩

每 2×2 区域只保留最大值，步长=2。

32×32 → 16×16，图变小一半，但每个区域最显著的特征被保留。

形状 = [64, 16, 16, 16]
⬇

🔍 第4步：第二个 Conv2d(16→32, kernel=3) + ReLU + MaxPool

和第一步原理完全一样，只是：

  • 输入从 3 通道变成了 16 通道（不是 RGB 了，是上一层的特征图）
  • 输出 32 个特征图
  • 每个核现在是 16 通道 × 3×3 = 144 个参数
直觉：第一层卷积从像素里找"低级模式"（边缘、颜色、纹理）， 第二层卷积在"低级模式"之上组合出"中级模式"（耳朵形状、轮子、眼睛）。 层越深，看到的模式越抽象。
形状变化: [64,16,16,16] → Conv → [64,32,16,16] → ReLU → 不变 → MaxPool → [64,32,8,8]
⬇

📏 第5步：Flatten() — 拉平

把 3D 特征图 排成一排，丢掉空间结构，准备喂给全连接层。

形状: [64,32,8,8] → [64, 2048] （32×8×8 = 2048）
⬇

🎯 第6步：两个 Linear 层 — 决策

Linear(2048 → 128)：压缩信息

Y₁ = X · W₁ + b₁

[64,2048] × [2048,128] + [128] = [64,128]

2048 个特征值通过矩阵乘法"融合"成 128 个高层语义概念。再 ReLU 激活。

Linear(128 → 10)：输出分数

Y₂ = Y₁ · W₂ + b₂

[64,128] × [128,10] + [10] = [64,10]

每张图得到 10 个分数——分数最高的那个就是模型的最终预测。

第1张图输出：[8.1, 1.2, -0.5, 3.1, 0.2, -1.0, 0.8, -0.3, 2.1, 4.0]
                 ↑ 最高 → 预测为 类别 0 = 猫

⚙️ Adam vs SGD 优化器参数详解

这两个优化器做同一件事：根据梯度更新模型参数。区别在于怎么更新。

共同公式（每次更新参数）：

θ_new = θ_old — 学习率 × 更新方向

关键在于 "更新方向"怎么算——SGD 和 Adam 的算法完全不一样。

SGD (带动量)

v = momentum × v + gradient

θ = θ — lr × v

理解：像骑自行车。当前梯度是"踩一脚"，但之前的速度（v）还在，继续往前滑。方向一致时越走越快，方向震荡时历史速度帮你平滑。

SGD 参数
lr学习率。SGD 对这个参数极其敏感。设太小（0.0001）→ 走不动。设太大（0.1）→ 震荡发散。典型值：0.01（比 Adam 大 10 倍）。因为 SGD 的梯度是"原始尺度"，没有自适应缩放。
momentum动量系数，几乎总是 0.9。0 = 纯 SGD（每步只根据当前梯度，走得很慢）。0.9 = 保留 90% 的历史速度。0.99 = 惯性太大，拐不了弯。经典值是 0.9，几十年没变过。
weight_decay每次更新把 weight 往 0 拉一点，防过拟合。典型值 1e-4~5e-4。
SGD 总结：简单、经典。但 lr 设错就崩。适合熟练工 + 大模型训练（很多 YOLO 的 paper 用 SGD 训的）。 你实验 3 用 SGD lr=0.001 → 54.3%，换成 lr=0.01+ momentum=0.9 才能发挥出来。

Adam (Adaptive Moment Estimation)

m = β₁ × m + (1-β₁) × gradient
← 一阶动量（梯度均值）
v = β₂ × v + (1-β₂) × gradient²
← 二阶动量（梯度方差）
θ = θ — lr × m / (√v + ε)
← 自适应学习率

理解：Adam 对每个参数都跟踪两个值——m（这个参数最近在往哪走）、v（走多快）。然后自动算出每个参数自己的"专属步长"。

梯度一直大的参数 → √v 大 → 步长自动缩小（别走过头）

梯度一直小的参数 → √v 小 → 步长自动放大（推一把）

Adam 参数
lr学习率。Adam 对这个参数不敏感。默认 0.001 在绝大多数场景都能跑。因为每个参数的步长被 √v 自动调整了，lr 本身更像一个"全局缩放系数"。
betas=(0.9, 0.999)(β₁, β₂)。β₁=0.9 控制"短期记忆"（类似 momentum），β₂=0.999 控制"长期速度统计"。几乎不用改，论文作者和全社区都用这个默认值。
eps=1e-8防除零的小常数，永远不改。
weight_decay和 SGD 一样。注意：Adam 建议用 AdamW 变体，它把 weight_decay 和自适应学习率解耦，效果更好。
Adam 总结：自适应、不挑 lr、开箱即用。入门首选。缺点：有时泛化不如精心调参的 SGD。 你实验 3 用 Adam lr=0.001 → 69.3%，对比 SGD lr=0.001 → 54.3%，差距就在 Adam 自动调整了步长。

你现在这个阶段怎么选：默认用 Adam/AdamW（lr=0.001）。如果效果可以就不折腾。 如果想复现某个经典 YOLO 配置 → 换成 SGD（lr=0.01, momentum=0.9, weight_decay=5e-4）。

核心心法：SGD 像是"所有人都走一样的步长，靠动量方向稳定"。 Adam 像是"每个参数有人扶着走，步子大小自动调"。 所以 Adam 对 lr 不敏感——你把 lr 写错了它也能自己修正；SGD 你写错 lr 直接训不动。
