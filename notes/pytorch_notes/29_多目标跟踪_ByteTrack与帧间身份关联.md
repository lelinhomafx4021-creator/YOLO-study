# 多目标跟踪：ByteTrack 与帧间身份关联

## 这一节在学习顺序里的位置

前面所有关卡（1-5）做的事情：

**每帧独立判断"这一帧有什么"。**

但真实场景不只看单帧。你想知道：

- 视频里出现了多少人（不是"这一帧有多少人"）
- 每个人出现了多久
- 同一个人走到屏幕另一边，还能认出他来

这就是**多目标跟踪（MOT）**。

---

## 先说最核心的一句话

**检测告诉你"这一帧有什么"，跟踪告诉你"帧与帧之间哪些是同一人"。**

如果只记一句，就记这句。

---

## 1. 检测和跟踪的区别

检测做的事情：

```
帧1: 有人 → 框A
帧2: 有人 → 框B
帧3: 有人 → 框C
```

**框A、框B、框C 是同一个人吗？检测不知道。**

跟踪做的事情：

```
帧1: 有人 → 框A (ID=1)
帧2: 有人 → 框B (ID=1)  ← 配对成功，是同一人
帧3: 有人 → 框C (ID=1)  ← 还是同一人
```

**现在知道了：有 1 个人，从帧 1 一直待到帧 3。**

---

## 2. 跟踪怎么做：三个步骤

每一帧，跟踪器做三件事：

### 第一步：检测

调 `model.predict()`，在这一帧找到所有框。

### 第二步：预测

对上一帧的每个 ID，用卡尔曼滤波猜"下一帧它会出现在哪"。

### 第三步：匹配

拿"这一帧的检测结果"和"预测的位置"做配对。

```
配对成功的 → ID 延续
检测多出的 → 新 ID（新人进场）
ID 没配上的 → 暂时保留，可能被遮挡了
```

---

## 3. 卡尔曼滤波：猜目标去哪了

卡尔曼滤波是跟踪的数学核心。不推公式，只记直觉：

```
小球在滚动:
  帧1: 球在 (100, 50)
  帧2: 球在 (110, 55)  ← 往右下方移动
  帧3: 球被手遮住了（检测没找到）

  卡尔曼说:
    "帧1→帧2 速度是 (10, 5)"
    "我预测帧3球在 (120, 60)"
    → 在附近搜索 → 找到了 → ID 没丢！
```

**卡尔曼滤波维护的是一个"状态"：位置 + 速度。每帧用位置更新速度，用速度预测下一帧位置。检测找到了就纠正，没找到就靠预测顶上。**

---

## 4. 匈牙利算法：最优配对

每帧给你一组"检测框"和一组"已追 ID"，怎么配对？

```
检测框: D1 D2 D3
已追ID: T1 T2 T3 T4

问: 每个 Di 应该跟哪个 Tj 配对？
```

匈牙利算法的做法：**算所有可能配对的总成本，选最小的那个。**

不是"谁离谁近就配谁"（那叫贪心），而是全局最优。

```
D1 离 T1 近、也离 T2 近
D2 只离 T1 近

贪心: D1 配 T1 → D2 没得配 → 差
匈牙利: D1 配 T2，D2 配 T1 → 总成本最低 → 好
```

**匈牙利算法就是找全局最优配对的算法。你不需要自己写，跟踪器内置了。**

---

## 5. ByteTrack 的核心创新

其他跟踪器的做法：

```
所有检测框 → 置信度 > 0.5 的才保留 → 去匹配 → 低分的直接扔
```

问题：遮挡严重时，有人的置信度掉到 0.4 → 被扔了 → ID 断了。

ByteTrack 的做法：

```
第一步: 高分框 (> 0.5) → 和已有 ID 匹配
第二步: 低分框 (0.3~0.5) → 也去匹配！给第二次机会
```

**ByteTrack 就做了一件事：低分检测框不扔，也拿去匹配。** 因为低分不一定是"没有目标"，可能只是"被挡了一部分"。

---

## 6. `model.track()` 和 `model.predict()` 的区别

```python
# 检测: 每帧独立
results = model.predict(frame, conf=0.25)

# 跟踪: 跨帧关联
results = model.track(frame, persist=True, tracker="bytetrack.yaml", conf=0.25)
```

track 内部先调 predict 做检测，然后跑卡尔曼 + 匈牙利做身份关联。

**返回值多了一个东西：`boxes.id`。**

```python
results[0].boxes.id   # tensor([1., 2., 1., 3.])
# 每个框的 ID 编号
# 同一目标在多帧中 ID 不变
```

`persist=True` 的作用：即使某一帧没检测到 ID=3，这个 ID 仍然保留在追踪列表里，不会立刻丢弃。

---

## 7. 记录每个 ID 的生命周期

```python
from collections import defaultdict
id_history = defaultdict(list)

# 每帧:
for tid in ids:
    id_history[tid].append(frame_count)   # "ID tid 在第 frame_count 帧出现过"
```

用 `defaultdict(list)` 的好处：键不存在时自动创建空列表，不用手动判断。

从 `id_history` 可以算出：

- `len(id_history)` — 视频里出现了多少个不同的人
- 每个 ID 出现了多少帧（待了多久）
- 最近 5 帧还在的 → 活跃的；超过 5 帧没出现 → 离开了

---

## 8. 卡尔曼滤波还能让框更准

不只是预测位置，还能平滑抖动。

```
检测器输出: 帧1 (100,50) 帧2 (110,55) 帧3 (130,40) ← 突然跳了

卡尔曼:
  "按前两帧的速度，这帧应该在 (120, 60)"
  "检测器说 (130, 40) — 我不太信"
  → 取一个加权平均: 大概 (124, 52)
```

**卡尔曼融合预测和检测：检测器稳定时信检测多一点，检测器抖动时信预测多一点。** 框比纯检测更平滑。

---

## 9. 跟踪的局限

1. **遮挡太久会丢 ID**: 卡尔曼靠速度预测，遮挡超过 10 帧后预测位置漂移，重出现时可能被分配新 ID

2. **人群密集会配错**: 匈牙利的"距离矩阵"在密集场景下有很多近似的距离，可能配对错误

3. **相机移动时需要运动补偿**: 本脚本假设摄像头静止。手持/车载需要先估计相机运动

---

## 10. 跟踪和跳帧的关系

你可能会想：跟踪能解决跳帧时框丢失的问题吗？

**能，但要自己做。**

跳帧时用卡尔曼预测框（不用跑检测）：

```
检测帧:  track → 检测 + ID + 纠正卡尔曼
跳过帧:  不跑检测，卡尔曼 predict() → 用预测位置画框
```

**卡尔曼预测就是给跳帧画框用的。** ByteTrack 内部的卡尔曼预测只用于"下一帧匹配时知道 ID 大概在哪"，不会自动给跳过的帧画框。你得自己用卡尔曼做这件事。

---

## 11. 这一节最该记住的 5 句话

1. **检测 = 每帧找框。跟踪 = 检测 + 身份关联**
2. **卡尔曼滤波 = 用历史位置预测未来位置，防遮挡丢失**
3. **匈牙利算法 = 给检测框和已追 ID 做全局最优配对**
4. **ByteTrack = 低分框不扔，给第二次机会匹配已追 ID**
5. **`model.track()` = `model.predict()` + `boxes.id`**

---

## 12. 学完这一节后你应该能回答的问题

1. 检测和跟踪的区别是什么
   - 检测: 单帧独立 — 告诉你"这一帧有什么"（框的坐标+类别），不知道帧间关联。每帧从零开始。
   - 跟踪: 帧间关联 — 告诉你"这些框里哪几个是同一个人"。给每个框一个 ID，同一目标跨帧 ID 不变。
   - 一句话: 检测 = 找框，跟踪 = 检测 + 身份关联。

2. 卡尔曼滤波在跟踪里做什么
   - 维护每个目标的"状态": 位置(cx,cy,w,h) + 速度(vx,vy)。检测帧: 用检测结果纠正状态（correct）。跳过帧/遮挡帧: 用速度外推位置（predict）。核心价值: 短暂遮挡时 ID 不丢，框还能靠预测位置继续画。

3. 匈牙利算法做什么，和贪心配对有什么区别
   - 匈牙利: 全局最优配对。N 个检测框配 M 个追踪 ID，算所有可能的 (N×M) 距离，找"总距离最小"的配法。
   - 贪心: 每次找最近的配，配完就排除 — 可能短视。例: D1 离 T1 近也离 T2 近，贪心配 D1→T1，D2 没得配；匈牙利配 D1→T2, D2→T1，总距离更小。
   - 你不需要自己写 — 跟踪器内置了。

4. ByteTrack 和其他跟踪器的区别在哪
   - 其他跟踪器: 低置信度框(conf<0.5)直接丢弃 → 人半遮挡时 confidence 降到 0.4 → ID 断了。
   - ByteTrack: 低分框也拿去匹配已追 ID → 被遮挡的人虽然分低，但位置和上一帧接近 → 能匹配成功 → ID 延续。核心创新: "低分不扔，给第二次机会"。

5. `model.track()` 比 `model.predict()` 多返回什么
   - predict() 返回: boxes.xyxy(坐标), boxes.conf(置信度), boxes.cls(类别)。
   - track() 额外返回: boxes.id — 每个框的跟踪 ID。同一目标在多帧中 ID 不变。
   - 内部: track = predict + 卡尔曼预测 + 匈牙利匹配 + ID 分配。

6. `persist=True` 是什么作用
   - 某帧没检测到某个 ID → 该 ID 在追踪列表里暂时保留，不会立刻丢弃。短暂遮挡(1~2帧)后重出 → ID 不变。不配时: 只要一帧没检测到，ID 立刻消失，回来变新 ID。

7. 卡尔曼滤波除了预测位置，还有什么用
   - 平滑框的抖动。检测器输出不稳定(连续帧框的位置轻微跳动)，卡尔曼融合"预测位置"和"检测位置"，取加权平均 → 框的轨迹更平滑，看起来不抖。权重自动决定: 检测稳定时多信检测，检测抖动时多信预测。

8. 从 `id_history` 能算出哪些统计
   - 总不同 ID 数: len(id_history)。活跃 ID 数: 最近 5 帧还在出现的。每个 ID 停留时长: len(frames) — 出现多少帧。某个 ID 的首次/最后出现帧号。可以算出: 场景流量（进出人数）、停留时间分布。

9. 跟踪怎么和跳帧配合使用
   - 检测帧: model.track() → 检测 + ID 关联 + 纠正卡尔曼。
   - 跳过帧: 不跑 track()，自己用卡尔曼 predict() 外推位置 → 直接画预测框（带 (pred) 标记）。
   - 效果: 每帧都有框，但只检测帧耗算力。代价: 跳过帧的框是"猜的"，目标突然变速时飘。

---

## 复习速答

- `检测`：这一帧里有什么。
- `跟踪`：这些框里谁是谁。
- `ByteTrack`：低分框也拿来匹配，减少 ID 断裂。
- `卡尔曼滤波`：预测下一帧位置并平滑抖动。
- `匈牙利算法`：做全局最优匹配。
