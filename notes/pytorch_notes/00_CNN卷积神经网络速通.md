# CNN 卷积神经网络速通

## 一句话

**CNN = 专门看图片的神经网络。用"滑动窗口"扫描图片，逐层从像素中提取特征，最后做出分类判断。**

## 1. 图片在电脑里是什么

```python
img.shape = (3, 32, 32)
#            3 个通道(RGB) × 32 高 × 32 宽
```


## 2. 卷积到底在干嘛

### 2.1 最容易犯的误解（纠正）

```
❌ 误解：核是 3×3 平面，在 3D 立方体上一个通道一个通道滑

✅ 真相：核是 3×3×3 立方体，"一口咬穿"3 个通道，
         27 个像素 × 27 个权重 → 一次算完
```

### 2.2 用放大镜理解

想象你拿一个 3×3 的小放大镜，**放大镜有 3 层镜片**（对应 RGB 三层）。

把它放在图片位置 (x,y) 上时，同时看到 3 个通道的 3×3 区域 = **27 个像素**。

这 27 个像素各自乘上放大镜的 27 个权重，求和 → 这个位置输出 1 个数。

**然后放大镜滑到下一个位置，再做一次。**

```
输入图（单通道 5×5 为例）：

┌───┬───┬───┬───┬───┐
│ 1 │ 2 │ 3 │ 0 │ 1 │        核（3×3）：
├───┼───┼───┼───┼───┤        ┌───┬───┬───┐
│ 4 │ 5 │ 6 │ 2 │ 3 │        │ 1 │ 0 │-1 │
├───┼───┼───┼───┼───┤        ├───┼───┼───┤
│ 7 │ 8 │ 9 │ 4 │ 5 │        │ 1 │ 0 │-1 │
├───┼───┼───┼───┼───┤        ├───┼───┼───┤
│ 0 │ 3 │ 5 │ 1 │ 2 │        │ 1 │ 0 │-1 │
├───┼───┼───┼───┼───┤        └───┴───┴───┘
│ 2 │ 1 │ 0 │ 3 │ 4 │
└───┴───┴───┴───┴───┘

核放在左上角 (0,0)：
  = 1×1 + 2×0 + 3×(-1)
  + 4×1 + 5×0 + 6×(-1)
  + 7×1 + 8×0 + 9×(-1)
  = 1 + 0 - 3 + 4 + 0 - 6 + 7 + 0 - 9
  = -6  → 输出图位置 (0,0) = -6

核右移一格到 (0,1)：
  = 2×1 + 3×0 + 0×(-1)
  + 5×1 + 6×0 + 2×(-1)
  + 8×1 + 9×0 + 4×(-1)
  = 2 + 0 + 0 + 5 + 0 - 2 + 8 + 0 - 4
  = 9  → 输出图位置 (0,1) = 9

...继续右移、下移，滑完所有位置...

5×5 的图，3×3 的核 → 输出 3×3 的特征图：

┌───┬───┬───┐
│-6 │ 9 │ -3│
├───┼───┼───┤
│-6 │ 9 │ -3│
├───┼───┼───┤
│ 6 │ 3 │ 0 │
└───┴───┴───┘
```

> 打开 `00_basics/cnn_visual.html` 点"自动播放"，看完整的逐格滑动动画。

**RGB 三通道就是：这个操作同时在 3 层上做，核深度也是 3，27 个权重对 27 个像素一次性算完。**

### 2.3 一个核滑完整张图 = 一张"特征图"

```
输入图（3×32×32） → 1 个 3×3×3 核滑一遍 → 输出 1 张特征图（32×32）
```

### 2.4 多个核 = 多张特征图

```
输入图（3×32×32）→ 16 个不同的核各滑一遍 → 16 张特征图（16×32×32）
```

**每个核的权重不同，产出的特征图就不同。** 核 1 可能学成"竖线检测器"，核 5 可能学成"红蓝过渡检测器"…… 这些权重是训练自动学出来的，不是你指定的。

## 3. ReLU — 负数归零

```python
relu = nn.ReLU()
```

```
[-2, 3, -1, 5] → ReLU → [0, 3, 0, 5]
```

**"没有这个特征"就干净归零，别留负数干扰后面的判断。**

---

## 4. MaxPool — 压缩图片，保留核心

```python
pool = nn.MaxPool2d(2)
```

每 2×2 取最大值，尺寸减半：

```
┌───┬───┐          ┌───┐
│ 3 │ 7 │          │   │
├───┼───┤   →      │ 7 │   只留最强的那个信号
│ 1 │ 2 │          │   │
└───┴───┘          └───┘
```

32×32 → 16×16。半个像素都没了，但"关键信号"还在。

## 5. 第二个卷积 — 在特征上再找特征

```
第一个 Conv2d(3, 16)： 从 RGB 像素里找低级特征（边缘、颜色、纹理）
第二个 Conv2d(16, 32)：在低级特征之上组合出中级特征（形状、局部图案）

第一层核：深度 = 3（RGB 通道）
第二层核：深度 = 16（上一层的 16 张特征图），所以是 [16×3×3] = 144 个参数
```

**层越深，看到的越抽象。** 浅层："这里有竖线"；中层："竖线 + 横线 = 一个角"；深层："角 + 角 + 角 + ... = 一只耳朵"。

---

## 6. Flatten — 拉平，丢掉空间结构

```
[32, 8, 8]  32 张 8×8 特征图
     ↓
32 张 × 64 格/张 = 2048 个数，按顺序排成一排

特征图0 (8×8): [a1, a2, ..., a64]
特征图1 (8×8): [b1, b2, ..., b64]
...
特征图31 (8×8): [z1, z2, ..., z64]

Flatten → [a1, a2, ..., a64, b1, b2, ..., b64, ......, z1, z2, ..., z64]
            └─── 64 ──┘ └─── 64 ──┘         └─── 64 ──┘
                           总长 = 2048
```

**这 2048 个数不是参数，是特征值。** 每张图算出来的都不一样。它们是"这份图的体检报告"，全连接层读它做分类。

> 常见混淆：`[32, 32, 8, 8]` 里两个"32"不是一回事。第一个是 batch（32 张图片），第二个是通道数（32 张特征图）。

---

## 7. Linear — 矩阵乘法做决策

```
[2048] × [2048, 128]  + [128]  → [128]    ← 2048 个特征融合成 128 个高层概念
[128]  × [128, 10]    + [10]   → [10]     ← 128 个概念 → 10 个类别的分数
```

## 8. 完整代码 + 每行注释

```python
class SimpleCNN(nn.Module):
    def __init__(self):
        super().__init__()
        # 前半段：提特征
        self.features = nn.Sequential(
            # Conv2d(入通道, 出通道, 核尺寸, 补零)
            # 入 3 (RGB) → 出 16 张特征图。核实际是 3×3×3=27个权重+1个bias
            nn.Conv2d(3, 16, kernel_size=3, padding=1),   # 448 个参数
            nn.ReLU(),                       # 负数归 0
            nn.MaxPool2d(2),                 # 32×32 → 16×16

            # 入 16 张特征图 → 出 32 张。核是 16×3×3=144个权重+1个bias
            nn.Conv2d(16, 32, kernel_size=3, padding=1),  # 4,640 个参数
            nn.ReLU(),
            nn.MaxPool2d(2),                 # 16×16 → 8×8
        )
        # 后半段：做判断
        self.classifier = nn.Sequential(
            nn.Flatten(),                    # [32,8,8] → [2048]
            nn.Linear(32*8*8, 128),          # 262,272 个参数（最多的！）
            nn.ReLU(),
            nn.Linear(128, 10),              # 1,290 个参数
        )

    def forward(self, x):
        x = self.features(x)      # 看图 → 提特征
        x = self.classifier(x)    # 读特征报告 → 做分类
        return x
```

## 9. 完整形状流水线（batch=32 为例）

```
输入               [32, 3, 32, 32]     32 张 RGB 图

Conv1(3→16)        [32, 16, 32, 32]    16 个核 × 每核 27 个参数
ReLU               不变
Pool1               [32, 16, 16, 16]   高宽减半

Conv2(16→32)       [32, 32, 16, 16]    32 个核 × 每核 144 个参数
ReLU               不变
Pool2               [32, 32, 8, 8]     高宽再减半
                                        ↑ batch ↑ 通道 ↑ 高 ↑ 宽
Flatten             [32, 2048]         32×8×8=2048 个特征值
Linear(2048→128)    [32, 128]          矩阵乘
ReLU               [32, 128]
Linear(128→10)      [32, 10]           每张图 → 10 个分类分数
```

## 10. 常见误区清单（踩过的坑）

| 误区 | 纠正 |
|------|------|
| 核是 3×3 平面 | 核是 3×3×通道数 的立方体，一次"咬穿"所有通道 |
| 核一个通道一个通道滑 | 所有通道同时算，27 个像素一次性加权求和 |
| 16 个核输出也一样 | 权重不同 → 特征图不同；移动路径相同，但"看问题的角度"不同 |
| Flatten 出来的数是参数 | **不是参数，是特征值**。每张图算出来的都不一样 |
| `[32,32,8,8]` 两个 32 分不清 | 第一个 32 = batch，第二个 32 = 通道数（特征图张数） |
| 每个核共享一个 bias | **每核一个专属 bias**，Conv(3→16) 有 16 个 bias |
| Linear 只有 weight 没有 bias | 每个输出神经元有 1 个专属 bias |

## 11. 参数 vs 特征值

```
参数（模型自己的）        特征值（图算出来的）
┌──────────────┐        ┌──────────────┐
│ 固定不变      │        │ 每张图都不同   │
│ 268,650 个   │        │ 这次 forward   │
│ 保存在 .pt   │        │ 临时算出来的   │
│ 训练时被梯度更新│      │ 特征图的输出   │
│ 训完就冻结    │        │ 和 Flatten 之后  │
└──────────────┘        └──────────────┘
```

## 12. 最该记住的几句话

1. **卷积 = 3D 小窗口在图上滑动，一次"啃"27 个像素，输出 1 个数**
2. **16 个核 = 16 种不同的特征图，权重是自动学出来的**
3. **池化 = 只保留最大值，缩小图片**
4. **Flatten = 重新排列，丢掉空间，变成特征清单**
5. **全连接 = 读特征清单，矩阵乘法做分类决策**
