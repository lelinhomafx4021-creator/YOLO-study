# DINO / DINOv2：自监督视觉 Transformer

## 这一节在学习顺序里的位置

前面你已经知道：

- ViT 用 Transformer 做图像分类，但需要大量标注数据
- DETR 用 Transformer 做目标检测，但训练慢

DINO（2021，Facebook）和 DINOv2（2023）解决的是另一个问题：

**不用标注数据，也能训练出好的视觉特征。**

---

## 先说最核心的一句话

**DINO 让模型自己学：同一张图的不同裁剪应该输出相似的特征，不同图应该输出不同的特征。不需要任何标签。**

---

## 1. 什么是自监督学习

普通监督学习：

```
图片 + 标签 → 模型 → 预测
猫的图 + "猫" → 模型 → 学到"这是猫"
```

需要人工标注。ImageNet 有 130 万张标注图，花了大量人力。

自监督学习：

```
图片（无标签）→ 模型 → 自己学特征
猫的图 → 模型 → 学到"这张图的特征是 [0.23, -1.45, ...]"
```

**不需要标签。模型从数据本身的结构中学习。**

---

## 2. DINO 的核心想法

DINO 的训练目标很简单： [看 Student/Teacher 可视化](dino_student_teacher_visual.html)

```
同一张图的不同裁剪 → 特征应该相似
不同图的裁剪 → 特征应该不同
```

具体做法：

```
一张猫的图
  ↓
裁剪出两个版本:
  大裁剪 (global): 看到猫的大部分
  小裁剪 (local):  只看到猫的一小块

Student 网络: 处理大裁剪 → 输出特征
Teacher 网络: 处理小裁剪 → 输出特征

训练目标: Student 的输出和 Teacher 的输出应该一致
```

**Student 和 Teacher 是同一个网络的不同版本。Teacher 是 Student 的指数移动平均（EMA）。**

---

## 3. 为什么需要两个网络

如果只有一个网络：

```
网络(大裁剪) → 特征A
网络(小裁剪) → 特征B
训练: A 应该等于 B
```

问题：网络会学到"不管输入什么，都输出一样的特征"。这样 Loss=0，但什么都没学到。

DINO 用两个网络避免这个问题：

```
Student(大裁剪) → 特征A
Teacher(小裁剪) → 特征B
训练: A 应该接近 B

但 Teacher 不直接更新！Teacher = EMA(Student)
→ Student 在努力追赶 Teacher
→ Teacher 比 Student 稳定
→ 不会坍缩到"所有输出都一样"
```

**Student 在努力学习，Teacher 是 Student 的"慢版本"，提供稳定的目标。**

---

## 4. DINO 学到了什么

DINO 训练完后，特征质量极高。可视化注意力图：

```
监督学习的 ViT:
  注意力分散，不太聚焦在物体上

DINO 训练的 ViT:
  注意力精准地集中在物体轮廓上！
  即使从没见过这个类别的标注，也知道"物体在这里"
```

**DINO 学到的特征天然包含了物体的边界信息。不需要任何分割标注，就能做前景/背景分离。**

---

## 5. DINOv2：更强的自监督

DINOv2（2023）在 DINO 基础上做了三个改进：

### 5.1 更多数据

```
DINO:  ImageNet (130 万张)
DINOv2: 1.42 亿张图（从互联网收集的大量无标注图片）
```

### 5.2 更大的模型

```
DINO:  ViT-B (8600 万参数)
DINOv2: ViT-g (11 亿参数)
```

### 5.3 更好的训练策略

混合了 DINO（自蒸馏）和 iBOT（掩码图像建模）两种自监督目标。

**DINOv2 的特征质量达到了"通用视觉特征"的水平——不微调就能做分类、分割、检索、深度估计。**

---

## 6. DINOv2 有多强

```
任务                    监督ViT    DINOv2
图像分类 (ImageNet)      81%       86%   ← 不微调就超过监督学习
语义分割 (ADE20K)        49        52
单目深度估计              0.11      0.07  ← 越低越好
图像检索                  中等       顶尖
```

**DINOv2 不微调就能做很多任务。它的特征是"通用视觉特征"，适用于几乎所有视觉任务。**

---

## 7. 和你做过的项目的关联

你做的是安全帽检测（有标注数据）。DINO 的用法：

```
场景1: 标注数据少
  → 先用 DINO 在无标注图片上预训练
  → 再用少量标注数据微调
  → 比从零训练效果好很多

场景2: 特征提取
  → 用 DINOv2 提取图片特征
  → 用简单的分类头（Linear）做分类
  → 不需要训练整个网络

场景3: 目标检测
  → 用 DINOv2 做 backbone
  → 比 ImageNet 预训练的 backbone 特征质量更高
```

4. **DINOv2 用 1.42 亿张图训练，特征质量达到了"通用视觉特征"水平**

## 9. 学完这一节后你应该能回答的问题

**Q1: 自监督学习和监督学习的区别是什么**

A: 监督学习 = 图片 + 标签 → 模型学。自监督学习 = 只有图片（无标签）→ 模型从数据本身的结构中学习。不需要人工标注。

---

**Q2: DINO 的训练目标是什么**

A: 同一张图的不同裁剪应该输出相似的特征，不同图应该输出不同的特征。不需要任何标签。

---

**Q3: 为什么需要 Student 和 Teacher 两个网络**

A: 如果只有一个网络，会学到"不管输入什么，都输出一样的特征"（坍缩）。Student 在学习，Teacher 是 Student 的 EMA（指数移动平均），提供稳定的目标，防止坍缩。

---

**Q4: Teacher 是怎么更新的**

A: Teacher = EMA(Student)。每一步 Student 更新后，Teacher 按公式缓慢跟进：Teacher = 0.996 × Teacher + 0.004 × Student。Teacher 比 Student 稳定。

---

**Q5: DINO 学到的特征有什么特殊之处**

A: 注意力图精准地集中在物体轮廓上。即使从没见过这个类别的标注，也知道"物体在哪里"。特征天然包含物体的边界信息。

---

**Q6: DINOv2 比 DINO 改进了什么**

A: 更多数据（1.42 亿张 vs 130 万张）、更大的模型（11 亿参数 vs 8600 万）、更好的训练策略（混合 DINO + iBOT）。

---

**Q7: DINOv2 的特征能用在哪些任务上**

A: 不微调就能做分类、分割、检索、深度估计。是"通用视觉特征"，适用于几乎所有视觉任务。

---

## 复习速答

- `自监督`：不靠人工标签学特征。
- `DINO`：用 Student/Teacher 防止特征坍缩。
- `Teacher`：Student 的 EMA 版本，更稳定。
- `DINOv2`：数据更多、模型更大、特征更强。
- `特征用途`：分类、分割、检索、深度估计都能用。
