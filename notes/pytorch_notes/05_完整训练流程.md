# 完整训练流程

## 这一节在学习顺序里的位置

我们现在已经走到这里：

1. `PyTorch 是什么`
2. `Tensor 是什么`
3. `什么是训练`
4. `什么是模型`
5. `完整训练流程`

前面两节你已经知道：

- 训练的本质是不断更新参数，让 `loss` 下降
- 模型是一个带参数的函数

那这一节要做的事情，就是把这些概念真正串起来：

**一轮标准的 PyTorch 训练，到底是怎么跑的。**

这一节非常关键，因为它是后面所有高层框架的共同骨架。

---

## 先说结论：训练循环的骨架

一个最标准的训练循环，通常长这样：

```python
for epoch in range(num_epochs):
    model.train()
    for x_batch, y_batch in dataloader:
        optimizer.zero_grad()
        pred = model(x_batch)
        loss = loss_fn(pred, y_batch)
        loss.backward()
        optimizer.step()
```

如果你先只背一段代码，先背这一段。

因为它几乎就是所有深度学习训练代码的母版。

---

## 1. 为什么叫“完整训练流程”

很多时候我们会单独讲某一个点：

- 什么是 `loss`
- 什么是 `grad`
- 什么是 `optimizer`
- 什么是 `batch`

但真正训练时，这些概念不是分开工作的，而是连成一条线：

```text
Dataset -> DataLoader -> batch
-> model 前向传播
-> loss 计算误差
-> backward 反向传播
-> optimizer 更新参数
-> 下一个 batch
```

所以“完整训练流程”不是新概念，而是把你前面学的东西接起来。

---

## 2. 第一步：准备数据

训练不能凭空发生，先得有数据。

在 PyTorch 里，这一步通常由两部分构成：

- `Dataset`
- `DataLoader`

### 2.1 Dataset 是什么

`Dataset` 的职责是：

**告诉程序：数据从哪来，单个样本怎么取。**

最常见的最小写法：

```python
from torch.utils.data import Dataset

class MyDataset(Dataset):
    def __init__(self, xs, ys):
        self.xs = xs
        self.ys = ys

    def __len__(self):
        return len(self.xs)

    def __getitem__(self, idx):
        return self.xs[idx], self.ys[idx]
```

这里有两个必须实现的方法：

- `__len__`：数据集有多少样本
- `__getitem__`：怎么取第 `idx` 个样本

你可以把 `Dataset` 理解成：

**单个样本的读取规则。**

### 2.2 DataLoader 是什么

`DataLoader` 的职责是：

**把 Dataset 里的单个样本，按 batch 打包送给训练循环。**

例如：

```python
from torch.utils.data import DataLoader

dataloader = DataLoader(
    dataset,
    batch_size=16,
    shuffle=True
)
```

这里：

- `batch_size=16`：每次取 16 个样本
- `shuffle=True`：每轮打乱顺序

你可以把它理解成：

**训练循环的数据供应器。**

---

## 3. 第二步：定义模型

训练时必须先有模型。

最小例子：

```python
import torch.nn as nn

class MyModel(nn.Module):
    def __init__(self):
        super().__init__()
        self.fc = nn.Linear(4, 1)

    def forward(self, x):
        return self.fc(x)
```

这里：

- `__init__` 定义模型里有哪些层
- `forward` 定义数据怎么流动

训练时真正被改的参数，就在这些层里面。

---

## 4. 第三步：定义损失函数和优化器

模型只是负责预测。

训练还需要另外两个角色：

- `loss function`
- `optimizer`

### 4.1 损失函数

损失函数负责：

**衡量模型预测和真实答案差多少。**

比如分类常见写法：

```python
loss_fn = nn.CrossEntropyLoss()
```

回归常见写法：

```python
loss_fn = nn.MSELoss()
```

训练时会这样用：

```python
loss = loss_fn(pred, target)
```

这一步之后，我们才有了一个可以被优化的数字：`loss`。

### 4.2 优化器

优化器负责：

**根据梯度真正更新模型参数。**

最常见写法：

```python
optimizer = torch.optim.SGD(model.parameters(), lr=0.01)
```

或者：

```python
optimizer = torch.optim.Adam(model.parameters(), lr=0.001)
```

这里最重要的是两点：

- `model.parameters()`：告诉优化器要管哪些参数
- `lr`：学习率，控制每次更新幅度

---

## 5. 第四步：进入训练循环

现在前置准备齐了：

- 有数据
- 有模型
- 有损失函数
- 有优化器

接下来就是完整训练循环。

一个更完整、能读懂的版本如下：

```python
for epoch in range(num_epochs):
    model.train()

    for x_batch, y_batch in dataloader:
        optimizer.zero_grad()

        pred = model(x_batch)
        loss = loss_fn(pred, y_batch)

        loss.backward()
        optimizer.step()
```

这一段代码里，每一行都有明确职责。

---

## 6. `model.train()` 是干什么的

这一句很多人会机械记，但没真懂。

```python
model.train()
```

它的作用是：

**把模型切换到训练模式。**

这件事只会影响某些层，比如：

- `Dropout`
- `BatchNorm`

不是每一层都会受影响，但训练时最好形成习惯：进入训练循环前先写它。

### 6.1 为什么训练模式重要

因为训练和评估时，有些层行为不一样。

例如：

- `Dropout` 训练时会随机丢弃部分单元
- `BatchNorm` 训练时会根据当前 batch 统计均值方差

所以：

**训练前用 `model.train()`，验证/推理前用 `model.eval()`。**

---

## 7. `optimizer.zero_grad()` 为什么要先写

这一句非常关键：

```python
optimizer.zero_grad()
```

它的作用是：

**清空上一次留下来的梯度。**

### 7.1 为什么要清

因为 PyTorch 默认是：

**梯度累加**

不是自动覆盖。

也就是说，如果你不清空：

- 上一个 batch 的梯度还在
- 这一个 batch 的梯度又加上去

那梯度就混了。

### 7.2 顺序为什么是先清再算

标准推荐模板里，一般写成：

```python
optimizer.zero_grad()
pred = model(x_batch)
loss = loss_fn(pred, y_batch)
loss.backward()
optimizer.step()
```

这样可以保证：

**每一次 `backward()` 计算的都是干净的新梯度。**

补一个小点：

PyTorch 官方教程里也有把 `zero_grad()` 放在 `step()` 后面的写法，只要保证“下一次 `backward()` 前梯度已经清空”就能工作。但在学习阶段，最推荐你固定成上面这个顺序，最不容易乱。

---

## 8. `pred = model(x_batch)` 是前向传播

```python
pred = model(x_batch)
```

这一句的作用是：

**让模型根据当前参数，对这一批输入做预测。**

这一步叫：

**前向传播 forward**

注意：

- 这一步会产生预测结果
- 但这一步还没有更新参数

所以前向传播只是训练的一部分，不是训练全部。

---

## 9. `loss = loss_fn(pred, y_batch)` 是计算误差

```python
loss = loss_fn(pred, y_batch)
```

这一句的作用是：

**把预测和真实答案做比较，得到一个“错多少”的数字。**

比如分类任务里：

- `pred` 是各类别得分
- `y_batch` 是真实标签

损失函数会把这两者变成一个标量 `loss`。

这个数越大：

- 说明错得越多

这个数越小：

- 说明预测越接近目标

---

## 10. `loss.backward()` 是反向传播

```python
loss.backward()
```

这一句的作用是：

**从 loss 开始反向传播，自动计算模型参数的梯度。**

也就是：

- 哪些参数该变大
- 哪些参数该变小
- 应该改多少

梯度算完后，会被放进对应参数的 `.grad` 里。

所以你可以记成：

- `forward`：算预测
- `loss`：算误差
- `backward`：算梯度

---

## 11. `optimizer.step()` 才是真正更新参数

```python
optimizer.step()
```

这一句经常被低估，但它非常关键。

它的作用是：

**根据刚才算好的梯度，真正修改模型参数。**

也就是说：

- `loss.backward()` 只是把修改建议算出来
- `optimizer.step()` 才是实际动手改参数

这也是为什么我们一直强调：

**`backward()` 和 `step()` 不是一回事。**

---

## 12. 一个 batch 到底发生了什么

把前面压缩成一次完整的 batch 训练：

```text
拿到一批数据
-> 清空旧梯度
-> 前向传播得到预测
-> 计算 loss
-> backward 计算梯度
-> step 更新参数
-> 这一批结束
```

这是训练的最小完整单位。

每处理完一个 batch，参数都会更新一次。

---

## 13. 一个 epoch 到底发生了什么

一个 epoch 的意思是：

**整个训练集完整过一遍。**

比如：

- 数据集有 1000 张图
- `batch_size = 100`

那一个 epoch 里会有：

- 10 个 batch
- 也就是 10 次参数更新

所以：

- `batch` 是一步一步训
- `epoch` 是整套数据完整走一遍

---

## 14. 为什么训练一定是双层循环

训练代码通常长这样：

```python
for epoch in range(num_epochs):
    for x_batch, y_batch in dataloader:
        ...
```

为什么要双层？

### 外层 `epoch`

控制：

- 把整个数据集看几遍

### 内层 `batch`

控制：

- 每次拿多大一批数据
- 每次完成一次参数更新

所以双层循环不是语法习惯，而是训练逻辑决定的。

---

## 15. 训练时为什么经常要把数据和模型放到同一个设备

如果你用 GPU 训练，常见写法是：

```python
device = torch.device("cuda" if torch.cuda.is_available() else "cpu")
model = model.to(device)
```

然后 batch 也要转过去：

```python
x_batch = x_batch.to(device)
y_batch = y_batch.to(device)
```

原因很简单：

**模型和数据必须在同一个设备上，才能一起计算。**

否则会报错。

所以完整训练里，通常还会加上“迁移到 device”这一步。

---

## 16. 验证和训练有什么区别

训练不是唯一流程，通常每个 epoch 后面还会做验证。

验证时最关键的不同是：

- 不更新参数
- 不计算梯度

常见写法：

```python
model.eval()
with torch.no_grad():
    for x_batch, y_batch in val_loader:
        pred = model(x_batch)
        loss = loss_fn(pred, y_batch)
```

### 16.1 `model.eval()`

作用是：

**把模型切到评估模式。**

主要影响：

- `Dropout`
- `BatchNorm`

### 16.2 `torch.no_grad()`

作用是：

**关闭梯度计算。**

这样做的好处是：

- 更省显存
- 更省计算
- 验证/推理更合适

所以：

- 训练时：`model.train()`
- 验证时：`model.eval()` + `torch.no_grad()`

---

## 16.3 `eval` 和 `val` 不是一回事

这个点特别容易混。

很多人会把：

- `model.eval()`
- `val`

当成同一个东西，但它们其实不是一个层级的概念。

### `model.eval()`

这是：

**PyTorch 里模型的模式切换函数。**

它的作用是：

- 把模型切到评估模式
- 让 `Dropout`、`BatchNorm` 这些层按“验证/推理”的方式工作

所以 `eval()` 只是在做一件事：

**切模式**

它本身不会：

- 自动跑验证集
- 自动计算指标
- 自动打印结果

### `val`

`val` 更像是一个完整阶段，意思是：

**拿验证集跑一遍，看看模型效果。**

它通常包含下面这些动作：

1. `model.eval()`
2. `with torch.no_grad():`
3. 遍历 `val_loader`
4. 计算验证 loss 或指标
5. 汇总结果

所以：

- `eval()` 是“切到验证模式”
- `val` 是“完整验证流程”

### 一句话区分

你可以直接这样记：

- `train()` / `eval()`：是模型模式
- `train阶段` / `val阶段`：是完整流程

---

## 16.4 训练、验证、推理三者代码对照

把这三个阶段放一起，你会更容易看懂它们的区别。

### 训练阶段

```python
model.train()

for x_batch, y_batch in train_loader:
    x_batch = x_batch.to(device)
    y_batch = y_batch.to(device)

    optimizer.zero_grad()
    pred = model(x_batch)
    loss = loss_fn(pred, y_batch)
    loss.backward()
    optimizer.step()
```

特点：

- 要算梯度
- 要更新参数
- 用 `model.train()`

### 验证阶段

```python
model.eval()

val_loss = 0.0

with torch.no_grad():
    for x_batch, y_batch in val_loader:
        x_batch = x_batch.to(device)
        y_batch = y_batch.to(device)

        pred = model(x_batch)
        loss = loss_fn(pred, y_batch)
        val_loss += loss.item()
```

特点：

- 不算梯度
- 不更新参数
- 要在整套验证集上统计结果
- 用 `model.eval()` + `torch.no_grad()`

### 推理阶段

```python
model.eval()

with torch.no_grad():
    x = image.to(device)
    pred = model(x)
```

特点：

- 只想要输出结果
- 不关心 loss
- 不更新参数
- 常常只处理一张图或一小批图

### 三者最短总结

- `train`：训练模型
- `val`：验证模型效果
- `predict / inference`：真正拿模型去用

## 17. 一个更完整的训练 + 验证模板

下面这段模板，已经很接近真实项目代码了：

```python
for epoch in range(num_epochs):
    model.train()
    train_loss = 0.0

    for x_batch, y_batch in train_loader:
        x_batch = x_batch.to(device)
        y_batch = y_batch.to(device)

        optimizer.zero_grad()
        pred = model(x_batch)
        loss = loss_fn(pred, y_batch)
        loss.backward()
        optimizer.step()

        train_loss += loss.item()

    model.eval()
    val_loss = 0.0

    with torch.no_grad():
        for x_batch, y_batch in val_loader:
            x_batch = x_batch.to(device)
            y_batch = y_batch.to(device)

            pred = model(x_batch)
            loss = loss_fn(pred, y_batch)
            val_loss += loss.item()

    print(
        f"epoch={epoch+1}, "
        f"train_loss={train_loss/len(train_loader):.4f}, "
        f"val_loss={val_loss/len(val_loader):.4f}"
    )
```

这段代码里最值得你记住的，不是每个字符，而是它背后的结构。

## 18. `loss.item()` 是干什么的

训练代码里你经常会看到：

```python
train_loss += loss.item()
```

这里的 `loss` 是一个 Tensor。

`loss.item()` 的作用是：

**把只包含一个值的 Tensor，取成 Python 里的普通数字。**

这样你才能：

- 打印
- 累加
- 记录平均 loss

所以 `item()` 不是训练核心，但它很常用。

## 19. 从底层看，Ultralytics 的 `train/val` 在帮你做什么

你现在已经用过：

```python
model.train(...)
model.val(...)
```

这些高层接口。

它们看起来像一句话搞定很多事，但底层本质上还是这一套：

### `train`

帮你做了：

- 构建数据集和 DataLoader
- 前向传播
- 计算 loss
- `backward()`
- `optimizer.step()`
- 记录训练结果

### `val`

帮你做了：

- 切换到评估模式
- 关闭梯度
- 跑验证集
- 统计指标，比如 `P / R / mAP`

也就是说：

**高层 API 只是把完整训练流程封装掉了，并没有改变训练本质。**

## 20. 初学者最容易犯的 8 个错误

### 20.1 忘记 `optimizer.zero_grad()`

会导致梯度累加，训练结果不稳定。

### 20.2 把 `backward()` 当成更新参数

其实 `backward()` 只算梯度，真正更新参数的是 `step()`。

### 20.3 训练和验证都用 `model.train()`

这样会让某些层表现不对，验证结果不稳。

### 20.4 验证时不加 `torch.no_grad()`

能跑，但会浪费显存和计算。

### 20.5 把 `epoch` 和 `batch` 混成一回事

一个 epoch 里通常有很多 batch。

### 20.6 只看训练 loss，不看验证 loss

这样容易看不出过拟合。

### 20.7 模型和数据不在同一个 device

会直接报错。

### 20.8 把“跑通代码”和“看懂流程”混成一回事

真正重要的是知道每一步在干嘛。

## 21. 这一节最该记住的几句话

### 21.1 训练循环骨架

**`zero_grad -> forward -> loss -> backward -> step`**

### 21.2 训练模式

**训练前用 `model.train()`，验证前用 `model.eval()`。**

### 21.3 验证模式

**验证/推理时通常配合 `torch.no_grad()`。**

### 21.4 batch 和 epoch

- `batch`：一次送进模型的一小批数据
- `epoch`：整个训练集完整看一遍

### 21.5 本质

**训练就是按 batch 反复更新参数，很多个 batch 组成一个 epoch，很多个 epoch 让模型逐渐学会任务。**

## 22. 一张完整训练流程图

```text
训练集
-> Dataset 定义单个样本怎么取
-> DataLoader 按 batch 打包
-> model.train() 切到训练模式
-> 对每个 batch:
   -> optimizer.zero_grad()
   -> model(x_batch) 前向传播
   -> loss_fn(pred, target) 计算损失
   -> loss.backward() 计算梯度
   -> optimizer.step() 更新参数
-> 一个 epoch 结束
-> model.eval() + torch.no_grad() 做验证
-> 重复多个 epoch
```
