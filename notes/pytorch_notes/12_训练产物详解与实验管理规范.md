# 🗂️ YOLO 训练产物详解与实验管理规范（告别 train-80 混乱）

当你在 VS Code 左侧看到 `train`, `train1`, `train2` 一路排到了 `train-80` 时，这其实说明你的实验已经陷入了“无序混乱”。你根本不记得 `train-45` 里跑的到底是什么超参数，哪个才是效果最好的模型。

本讲义将为你详细拆解**训练生成目录的秘密**，并教你一套**工业级/简历级的实验规范命名大法**。

---

## 🔍 第一部分：YOLO 训练结果目录文件大拆解

每次训练跑完，生成的文件夹（如 `runs/detect/your_experiment`）里会存放以下文件：

### 1. 📂 `weights/`（核心资产库）
* **`best.pt`**：**核心产物**。模型在验证集上表现最好（mAP最高）那一轮的权重。部署时只用它。
* **`last.pt`**：**备份断点**。最后一轮训练完的权重，用于训练意外中断时，通过 `resume=True` 接着训练。

### 2. 📝 `args.yaml`（超参备忘录）
* 里面记录了你启动这次训练时传入的**所有超参数**（比如你设的学习率 `lr0`、`batch`、`epochs`、`imgsz` 以及数据增强系数等）。
* **作用**：方便以后还原实验，防止“不记得当时是怎么训的了”。

### 3. 📊 `results.png` / `results.csv`（成绩单）
* 记录了每一轮（Epoch）下，训练集和验证集的 Loss 变化，以及 Precision, Recall, mAP50, mAP50-95 的数值。
* **面试查图**：
  * **过拟合识别**：如果 `train_loss` 一直往下降，但 `val_loss` 降到一半后开始掉头**往上升**，就说明模型发生**过拟合**（只学会了死记硬背）。
  * **不收敛识别**：如果 Loss 是一条平线，或者剧烈震荡甚至出现 `NaN`，说明学习率 `lr0` 设得太大了。

### 4. 🖼️ 混淆矩阵与评估曲线（面试必考评估指标）
* **`confusion_matrix.png`**：**混淆矩阵**。帮你一眼看出模型是不是经常“指鹿为马”。比如它会展示：“有 10% 的背景被误判成了安全帽”。
* **`BoxF1_curve.png`**：**F1 得分与置信度关系曲线**。
  * **实战妙用**：曲线最高点对应的 Confidence 值，就是**最完美的置信度阈值**（比如 0.42）。以后写推理代码时，直接把阈值设为 0.42，模型的综合表现最好。
* **`BoxPR_curve.png`**：**PR（Precision-Recall）曲线**。曲线下的面积就是 AP（平均精度）。面积越大，模型越牛。

### 5. 🖼️ 数据增强与预测对比图
* **`train_batch0 / 1 / 2.jpg`**：**数据增强排查图**。展示第一轮训练时，被 Mosaic、Mixup 各种魔改拼接后的图片。专门用来肉眼检查标注框有没有对齐。
* **`val_batch0_labels.jpg`**（验证集真实标注） vs **`val_batch0_pred.jpg`**（验证集模型预测）：
  * **作用**：把两张图并排打开，用肉眼看看到底是哪些奇形怪状的安全帽模型没认出来。

## 🛠️ 第二部分：工业级 YOLO 实验规范命名与保存机制

如果你在代码里不规定 `project` 和 `name`，YOLO 会默认保存在 `runs/detect/train/`，并且每次自动往后数（`train2`, `train3`...）。这在实际工程中是个灾难。

### 💡 规范实验命名的“黄金公式”

我们在调用 `model.train()` 时，应该使用 **项目名（Project） + 版本迭代名（Name）** 的规范组合：

```python
model.train(
    # 1. project：指定项目大目录，以【数据集/项目名称】命名
    project="runs/safety_helmet",  
    
    # 2. name：指定当前实验文件夹，以【模型大小_核心超参_实验版本】命名
    name="yolo11n_img640_baseline_v1",  
    
    # 3. exist_ok：允许在已存在的文件夹里训练，不往后累加后缀
    exist_ok=True, 
    
    # 其他超参数...
)
```

### 📋 推荐的实验命名命名规范示例：

| 实验轮次 | 实验目的 | 建议 name 命名 | 对应物理意义 |
| :--- | :--- | :--- | :--- |
| **第一轮** | 基线模型（Baseline） | `yolo11n_img640_baseline_v1` | 使用 11n 模型，640尺寸，跑初始默认参数。 |
| **第二轮** | 调大图片尺寸（探究分辨率影响） | `yolo11n_img960_scale_v2` | 使用 960 大图训练，探究对小目标安全帽的提升。 |
| **第三轮** | 调强数据增强（防过拟合） | `yolo11n_img960_strong_aug_v3` | 开启了强 Mosaic 和 Mixup 的第 3 版实验。 |
| **第四轮** | 换大模型跑精度（探究参数量影响）| `yolo11m_img640_baseline_v4` | 换用中等尺寸的 yolo11m 模型进行对比。 |

### 🚀 规范命名的巨大好处：
1. **一目了然**：打开 `runs/safety_helmet/`，里面的每个文件夹名都写清了这次实验用的是什么模型、多大尺寸、什么参数，再也不用瞎猜。
2. **极好吹水**：在简历里写”**规范管理了多达 10 轮的消融实验（Ablation Study），涵盖了不同图像尺度、模型规模与增强强度的对比，最终选出最优模型**”，逼格直接拉满！

## 🔨 第三部分：训练产物的实战用法

### 3.1 加载 best.pt 做验证/预测

```python
from ultralytics import YOLO

# best.pt = 验证集上 mAP 最高的那一轮权重
model = YOLO(“runs/safety_helmet/yolo11n_baseline_v1/weights/best.pt”)

# 验证
model.val(data=”data.yaml”)

# 预测
model.predict(source=”test.jpg”, save=True, conf=0.5)
```

### 3.2 从 last.pt 断点续训

```python
from ultralytics import YOLO

# 训了 50 轮被中断了 → 从 last.pt 接着训
model = YOLO(“runs/safety_helmet/yolo11n_baseline_v1/weights/last.pt”)
model.train(data=”data.yaml”, epochs=100, resume=True)
#                                          ↑ resume=True 从断点续
```

### 3.3 读取 args.yaml 看当初的配置

```python
import yaml

with open(“runs/safety_helmet/yolo11n_baseline_v1/args.yaml”) as f:
    config = yaml.safe_load(f)

print(f”当时用的 lr: {config['lr0']}”)
print(f”当时用的 batch: {config['batch']}”)
print(f”当时用的 imgsz: {config['imgsz']}”)
```

### 3.4 你的实际训练产物对照

```
你的 baseline 训练：runs/safety_helmet/yolo11n_baseline_v1/

├── weights/
│   ├── best.pt     ← 第 ? 轮 mAP 最高（跑 10_error_analysis.py 找出来）
│   └── last.pt     ← 第 50 轮（最后）
│
├── results.csv     ← 每轮的 Loss/P/R/mAP 数值（10_error_analysis.py 吃这个）
├── results.png     ← Ultralytics 自动画的曲线图
│
├── confusion_matrix.png         ← 混淆矩阵：安全帽 vs 没戴帽 vs 背心
├── BoxPR_curve.png              ← P-R 曲线
├── BoxF1_curve.png              ← F1 曲线 → 找最佳 conf 阈值
│
├── train_batch0.jpg             ← 看数据增强把训练图魔改成什么样
├── val_batch0_labels.jpg        ← 验证集真实标注
└── val_batch0_pred.jpg          ← 验证集模型预测 → 打开看漏了哪些
```

### 3.5 部署前的检查清单

```
☐ best.pt 的 mAP50 达标了吗？（跑 10_error_analysis.py 看）
☐ 打开 confusion_matrix.png → 哪个类别误检最多？
☐ 打开 val_batch0_pred.jpg → 肉眼检查漏检的是什么样子的目标
☐ args.yaml 里的配置你记得住吗？（imgsz、epochs、lr）
☐ 实验文件夹名是否说明了这次实验的关键参数？
```

---

## 复习速答

- `best.pt`：验证指标最好的权重。
- `last.pt`：最后一轮权重。
- `args.yaml`：保存训练时的超参数配置。
- `results.csv`：记录每轮训练的原始数值。
- `confusion_matrix`：看模型最爱把谁认成谁。
