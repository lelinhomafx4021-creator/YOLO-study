# 数据增强

## 这一节为什么重要

学完评估指标之后，一个很自然的问题就是：

**如果模型效果不够好，我们除了继续训练，还能做什么？**

答案之一就是：

**数据增强。**

它在目标检测里非常重要，因为很多模型的上限，很大程度上取决于：

- 数据是不是够多
- 数据是不是够多样
- 训练时模型有没有见过足够丰富的变化

所以这一节不是在讲“小技巧”，而是在讲模型泛化能力的重要来源。

---

## 先说最核心的一句话

**数据增强 = 在训练阶段对原始样本做随机变换，让模型看到更多变化版本的数据。**

它的目标不是“变魔术”，而是：

- 增加数据多样性
- 降低过拟合
- 提升泛化能力

---

## 1. 什么是数据增强

最常见的直觉例子：

原来你只有一张图片：

- 正常光照
- 正常方向
- 正常尺度

训练时你可以随机把它变成：

- 更亮一点
- 更暗一点
- 左右翻转
- 轻微旋转
- 放大或缩小

这样模型看到的就不再只是“这一张原图”，而是很多不同版本。

所以数据增强的本质是：

**不增加新采集数据的前提下，让训练分布变得更丰富。**

---

## 2. 为什么需要数据增强

这一步一定要讲透，不然数据增强很容易学成背参数。

### 2.1 减少过拟合

如果训练数据太少或太单一，模型容易：

- 死记训练集细节
- 对新图适应差

数据增强会打乱这种“死记硬背”的路径，逼模型去学更稳定的特征。

### 2.2 提高泛化能力

现实里的图像永远不是训练集那样整齐：

- 光照会变
- 目标大小会变
- 视角会变
- 背景会变

数据增强让模型提前在训练阶段就见过这些变化。

### 2.3 弥补数据采集成本高的问题

很多项目里，重新采大量数据很贵。

这时增强就是一种非常高性价比的手段。

但要注意：

**增强不能替代真正缺失的重要数据分布。**

它是放大已有信息，不是无中生有。

---

## 3. 数据增强是在什么时候发生的

很多初学者容易误会成：

- 先把增强后的图全部生成好，存进硬盘

但在常见训练框架里，尤其是 PyTorch / Ultralytics 这类流程中，更常见的是：

**在线数据增强。**

也就是：

- 训练读到样本时
- 临时随机做增强
- 送进模型
- 不一定把增强结果长期保存到硬盘

所以：

**增强通常发生在数据加载阶段，而不是模型里面。**

---

## 4. 数据增强并不是越强越好

这是最容易走偏的地方。

很多人会想：

- 既然增强有用，那是不是全开、全拉满最好？

答案不是。

因为数据增强必须满足一个前提：

**增强后的样本，仍然要符合你的真实业务场景。**

如果增强方式把图像变成了“不真实的错误样本”，模型反而会被教坏。

所以正确思路不是：

- 增强越多越好

而是：

- 增强要和任务场景匹配

---

## 5. 常见基础增强

先看最常见的一组基础增强。

### 5.1 左右翻转 `fliplr`

作用：

**把图像左右镜像。**

在目标检测里通常比较常见。

为什么有用：

- 让模型适应左右方向变化
- 提高方向鲁棒性

但不是所有任务都适合。

例如文字识别、车牌字符这类任务，左右翻转往往会破坏语义。

### 5.2 上下翻转 `flipud`

作用：

**把图像上下颠倒。**

它适不适合，特别依赖场景。

例如：

- 航拍、俯视类任务，有时还能接受
- 正常地面监控场景，很多时候并不合理

### 5.3 旋转 `degrees`

作用：

**对图片做一定角度的随机旋转。**

如果你的目标本来就会倾斜，这种增强通常更有帮助。

### 5.4 缩放 `scale`

作用：

**让目标随机变大或变小。**

这对目标检测非常重要，因为现实里目标距离远近差很多。

### 5.5 平移 `translate`

作用：

**让目标位置随机偏移。**

这能减少模型把“目标总在某个位置”的偏见学进去。

### 5.6 颜色扰动 `hsv_h / hsv_s / hsv_v`

作用：

**改变色调、饱和度、亮度。**

这对光照变化、天气变化、相机白平衡变化都很有帮助。

---

## 6. 为什么颜色增强常常很有用

因为现实里图像颜色变化非常常见：

- 白天和夜晚
- 晴天和阴天
- 室内和室外
- 不同摄像头

所以颜色增强的核心价值是：

**让模型不要过度依赖某一种固定光照和色彩条件。**

但也要小心：

如果类别本身高度依赖颜色语义，颜色增强过强可能会伤害任务。

这一点在强颜色判别任务上尤其要注意。

---

## 7. YOLO 里最有代表性的增强：Mosaic

`Mosaic` 是检测任务里最有代表性的增强方式。

### 7.1 Mosaic 做了什么：把 4 张图拼成 1 张

```
原始 4 张训练图：

图 A                     图 B
┌──────────┐            ┌──────────┐
│  🪖 大安全帽 │            │  小安全帽    │
│          │            │       🪖   │
└──────────┘            └──────────┘

图 C                     图 D
┌──────────┐            ┌──────────┐
│    ✨ 反光背心│          │ 没标注：纯背景 │
│   ✨      │            │          │
└──────────┘            └──────────┘

         ↓ Mosaic 拼接 ↓

┌─────────────┬─────────────┐
│   图 A 左上   │   图 B 右上   │
│   🪖         │        🪖    │
├─────────────┼─────────────┤
│   图 C 左下   │   图 D 右下   │
│   ✨   ✨     │  （纯背景）    │
└─────────────┴─────────────┘

结果：640×640 的新图，4 个场景混在一起
     → 小目标更多了（安全帽被缩小了）
     → 背景更复杂了（4 个不同场景）
     → 标注框自动跟着图片位置调整
```

### 7.2 为什么有效

- **制造小目标**：原图 640×640 的安全帽占半张图，拼成 4 合 1 后只有 1/4 区域 → 有效的”缩小”效果
- **场景多样性**：一张图里同时有室内、室外、灯光暗、灯光亮 → 模型被迫学通用特征
- **减少背景偏见**：如果 A 类总在白色背景出现，Mosaic 把 A 塞到各种背景里 → 模型不会根据背景猜类别

### 7.3 YOLO 里怎么用

```python
# YOLO 训练时 Mosaic 默认开启（前 90% epochs）
model.train(data=”data.yaml”, epochs=50)
# 内部自动：前 45 轮用 Mosaic，最后 5 轮关闭

# 手动控制 Mosaic 关闭时机
model.train(data=”data.yaml”, epochs=50, close_mosaic=10)
#                                                     ↑
#                             最后 10 轮关闭 Mosaic，用正常单张图训练

# 完全关掉 Mosaic（不推荐，除非数据集很小）
model.train(data=”data.yaml”, epochs=50, mosaic=0.0)
```

### 7.4 但不是没有代价

- 拼接缝是人为的——真实世界没有这种生硬拼接线
- 模型可能适应了拼接线 → 对真实单张图反而偏差
- 4 合 1 让图上每个目标变小 → 对大目标检测不一定好

**所以 Mosaic 很强大，但需要用 close_mosaic 收尾。**

---

## 8. `close_mosaic` 为什么存在

### 8.1 一张时间线看懂

```
Epoch:  1 ────────────────────────→ 40 ──────→ 50
         │                           │           │
增强强度  │████████ Mosaic 全开████████│ 关 Mosaic │
         │                           │           │
目标：    │ 开阔模型眼界                │ 回归真实   │
         │ 见各种场景和组合             │ 稳定收敛   │
         │ 制造小目标                  │ 定位更准   │
```

### 8.2 底层代码

```python
# 默认行为：YOLO 自动在前 90% epochs 开，后 10% 关
model.train(data="data.yaml", epochs=50)
# 内部等价于 close_mosaic = int(50 * 0.1) = 5
# 前 45 轮 Mosaic 开，最后 5 轮关

# 手动指定 close_mosaic
model.train(data="data.yaml", epochs=100, close_mosaic=15)
# 前 85 轮开，最后 15 轮关
```

### 8.3 为什么一定要关

```
开 Mosaic 时：
  模型学的是"拼接图"上的分布 → 位置有小偏移、边缘有拼接缝
  → 框的位置准度（mAP50-95）上不去，因为拼接缝干扰了框的定位

关 Mosaic 后：
  模型在正常单张图上训练 → 定位回到真实分布
  → mAP50-95 通常会在这最后几轮有小幅上涨
```

### 8.4 面试考点回顾

> **"为什么 YOLO 训练最后要关掉 Mosaic？"**

答：Mosaic 4 合 1 拼接会引入人为的拼接线和不自然的物体分布。前 90% 用它是为了开阔模型眼界、学鲁棒特征。最后 10% 关掉，让模型在真实单张图分布上稳定定位收敛，这 10% 通常能涨 mAP50-95（位置分）。

## 9. MixUp 是什么

`MixUp` 的做法是：

**把两张图和对应标签按一定比例混合。**

它的目的通常是：

- 增强鲁棒性
- 减少过拟合

在检测任务里，它没有 Mosaic 那么“招牌”，但在某些数据量偏少、过拟合明显的情况下，也可能有帮助。

不过：

**不是所有项目都必须开 MixUp。**

它是可选增强，不是通用默认最优解。

---

## 10. Copy-Paste 是什么

`Copy-Paste` 常见于：

- 检测
- 分割

它的思路是：

**把一个图中的目标拷出来，贴到另一张图里。**

它的价值通常在于：

- 增加目标出现位置的多样性
- 提高少样本类别的训练机会

但它更依赖具体实现和任务性质，不是你现在这阶段最核心必须吃透的增强。

## 11. Ultralytics 中常见增强参数怎么理解

下面这些参数，是你后面最常碰到的一组。

### 11.1 几何类

- `degrees`：旋转角度范围
- `translate`：平移幅度
- `scale`：缩放范围
- `shear`：错切变换
- `perspective`：透视变化

这一类增强主要影响：

**目标的几何形态和位置。**

### 11.2 翻转类

- `fliplr`
- `flipud`

这一类增强主要影响：

**方向与空间分布。**

### 11.3 颜色类

- `hsv_h`
- `hsv_s`
- `hsv_v`

这一类增强主要影响：

**颜色和光照鲁棒性。**

### 11.4 组合增强

- `mosaic`
- `mixup`
- `copy_paste`

这一类增强会：

**更强地改变图像整体分布。**

## 12. 怎么判断某种增强适不适合你的任务

一个非常实用的判断原则是：

**问自己：这种变化在真实部署场景里，会不会发生？**

如果答案是：

- 经常发生

那这种增强往往更合理。

如果答案是：

- 基本不会发生

那这种增强就要谨慎。

例如：

- 工地监控中，轻微亮度变化合理
- 左右翻转通常也还合理
- 上下翻转很多时候就不合理

所以增强一定要回到场景。

---

## 13. 数据增强和标签为什么必须同步

目标检测和普通分类不一样。

分类里你翻一张图，标签通常还是这个类别。

但检测里如果你做：

- 翻转
- 缩放
- 平移
- 裁剪

对应的边界框坐标也必须一起变。

否则会发生：

- 图变了
- 框没变

那标签就废了。

所以检测任务的数据增强难点之一，就是：

**不只是改图像，还要同步改框。**

这也是为什么高层框架很省事。

---

## 14. 数据增强和指标的关系

你可以把增强理解成：

**用训练时更丰富的数据变化，去换验证时更稳定的指标表现。**

例如：

- 更丰富的小目标变化，可能帮助 `Recall`
- 更丰富的背景变化，可能帮助 `Precision`
- 更自然的多尺度训练，可能提升整体 `mAP`

但没有任何增强是保证“必涨分”的。

增强的本质是：

- 改训练分布
- 提升泛化概率

最终还是要回到验证集指标来判断有没有帮助。

---

## 15. 数据增强不是替代数据质量

这一句非常重要。

如果你数据本身有这些问题：

- 标注错误很多
- 类别严重不平衡
- 某些关键场景根本没拍到

那单靠增强通常救不了。

所以：

**增强可以放大好数据的价值，但不能替代缺失的关键数据分布。**

这也是为什么项目后期常常要结合：

- 增强
- 补数据
- 清洗标注

一起做。

---

## 16. 一个更实用的调参思路

不要一开始就同时乱调很多增强参数。

更推荐这样做：

### 16.1 先跑 baseline

先看不开太多花哨增强时，模型的基础表现如何。

### 16.2 针对问题加增强

如果你发现：

- 小目标漏检多

那就更关注：

- 多尺度
- Mosaic

如果你发现：

- 光照变化下表现差

那就更关注：

- `hsv_*`

### 16.3 一次改少量参数

否则你很难判断到底是哪种增强起了作用。

所以数据增强不是靠“全开”，而是靠“有针对性地加”。

## 17. 初学者最容易犯的 7 个错误

### 17.1 认为增强越强越好

不对，增强必须符合任务场景。

### 17.2 同时乱调很多增强

最后根本不知道哪一个有用。

### 17.3 忘了检测任务里框也要同步变

这是检测增强的核心难点之一。

### 17.4 用不符合场景的翻转

比如场景里几乎不可能上下颠倒，却开很强的 `flipud`。

### 17.5 只看训练 loss，不看验证指标

增强好不好，最终要看验证效果。

### 17.6 把增强当成补数据的万能药

增强不能创造真实缺失的关键样本。

### 17.7 训练后期还一直强增强

有时会影响模型后期稳定收敛。

## 20. 学完这一节后，你应该能回答的问题

1. 数据增强到底是什么
   - 对已有训练图片做随机变换（翻转、旋转、缩放、颜色扰动），生成"不同的"新图片，但标签也跟着变换。模型看到的是变种图片，不会死记原图。相当于免费扩了训练集。

2. 为什么它能帮助模型泛化
   - 训练集有限，覆盖不了真实场景的所有变化（光线、角度、距离）。增强可以模拟这些变化 → 模型在训练时见过更多"风格"的数据 → 真实场景下更不慌。本质是强制模型学习"真正稳定的特征"而不是"原图的特定像素排列"。

3. 为什么增强不是越多越强越好
   - 增强太猛 → 图片失真（如旋转 90° 的停车标志对检测没用，旋转 15° 可以）。检测任务中: 框的坐标必须跟着变换，空间变换太猛 → 框变形或截断 → 模型学到错标签。过度增强 = 引入噪声 = 准确率反而下降。

4. 检测任务里的增强为什么比分类更麻烦
   - 分类只需要标签(猫/狗)不变。检测需要框坐标也跟着变换 — 翻转/旋转/缩放后 x1y1x2y2 全都得重新算。增强错了 → 框歪到别的物体上 → 模型被错误标签训练 → 越训越差。所以 YOLO 的增强是精心设计的（Mosaic、hsv_h、fliplr 等专门做了坐标变换）。

5. Mosaic 和 `close_mosaic` 的作用是什么
   - Mosaic: 把 4 张图拼成一张 → 丰富背景、模拟小目标。训练前期(epochs 的前 90%)开。
   - close_mosaic: 训练最后 10% 回合关掉 Mosaic → 模型学到完整图片的模式，不依赖"4 张拼"的分布。如果不关，推理时喂的是正常单图，训练看到的全是 4 拼图，可能性能差。

6. 为什么增强效果最终要靠验证指标判断
   - 增强目标: 验证集上 mAP/acc 更高。增强手段和强度没有标准答案（数据不同 → 最佳增强不同）。唯一验证手段: 加增强 → 跑训练 → 看验证指标 → 更好就保留，更差就回退。不能凭感觉说"增强多一点应该会好吧"。

**调参优化。**

- 训练主线

**当指标不理想时，我们到底该先调什么、后调什么。**

## 复习速答

- `数据增强`：训练时对图片做随机变换。
- `作用`：增加样本多样性，减少过拟合。
- `检测任务难点`：图变了，框也要跟着变。
- `Mosaic`：把多张图拼成一张的强增强。
- `close_mosaic`：训练后期关闭强增强，让模型回到真实单图分布。
