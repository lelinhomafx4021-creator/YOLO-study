# 调参优化

## 这一节为什么重要

到这里，你已经知道了：

- 训练流程怎么跑
- 验证指标怎么看
- 数据增强在做什么

接下来最自然的问题就是：

**如果模型效果不够好，我们到底该先调什么、后调什么？**

这就是调参。

这一节不会只讲抽象概念，我会按最实用的顺序来：

1. 先讲调参到底在调什么
2. 再讲最重要的几个参数
3. 再讲怎么结合 `loss / Precision / Recall / mAP` 来调
4. 最后给你 `PyTorch` 和 `YOLO/Ultralytics` 的 API 示例

---

## 先说结论：调参不是乱试参数

**调参 = 根据训练现象和验证指标，有目标地调整超参数。**

这里的重点不是“参数越多越厉害”，而是：

**看到什么问题，优先改哪一类参数。**

所以调参不是玄学，更不是瞎撞。

---

## 1. 什么叫“超参数”

先把“参数”和“超参数”分开。

### 1.1 模型参数

模型参数是训练过程中自动学出来的，比如：

- `weight`
- `bias`

它们由：

- `loss.backward()`
- `optimizer.step()`

不断更新。

### 1.2 超参数

超参数是训练前或训练过程中由我们控制的设置，比如：

- `lr`
- `batch_size`
- `epochs`
- `imgsz`
- `optimizer`
- `weight_decay`

这些不是模型自己学出来的，而是你人为决定的。

所以：

**训练是在学参数，调参是在调超参数。**

---

## 2. 调参到底在解决什么问题

调参最常见是为了解决下面这些问题：

- `loss` 不降
- 训练太慢
- 验证效果差
- 过拟合
- 欠拟合
- 显存爆掉
- 小目标检测差
- 误检多 / 漏检多

所以你以后调参时，不要从“我想调哪个参数”出发，而要从：

**我现在遇到的主要问题是什么**

出发。

---

## 3. 最应该优先关心的 6 个超参数

你现在先不要被几十个参数吓到。

真正最先要会的，是这 6 个：

1. `epochs`
2. `batch_size`
3. `learning rate`
4. `imgsz`
5. `optimizer`
6. `weight_decay`

如果这 6 个你能看懂，已经够你做大部分基础优化了。

### 3.1 对照你的代码（08_mytestforcnn.py）

每个超参数在你的 CNN 代码里都有一行对应，你已经在用了，只是没意识到：

```python
# 你的 08_mytestforcnn.py 里的超参数位置：

epoch_num = 10                              # ← epochs（第118行）

trainloader = DataLoader(..., batch_size=64) # ← batch_size（第38行）

optimizer = Adam(model.parameters(), lr=0.001)  # ← lr（第116行）
#                                    lr=0.001 就是 learning rate

optimizer = torch.optim.Adam(...)           # ← optimizer 选的是 Adam（第116行）

# weight_decay 你没写，默认是 0             # ← weight_decay（没用到）
# imgsz CIFAR-10 固定 32×32，改不了         # ← imgsz（数据集决定）
```

**一句话**：你之前的训练其实已经在调参了，只是用的是"默认值"，没有刻意对比过。

---

## 4. `epochs` 是什么，怎么调

`epochs` 表示：

**整个训练集完整看多少遍。**

### 4.1 API 例子

#### PyTorch 里

```python
num_epochs = 50

for epoch in range(num_epochs):
    ...
```

#### Ultralytics 里

```python
model.train(data="data.yaml", epochs=50)
```

### 4.2 太少会怎样

- 训练还没学够
- `loss` 还在下降就结束了
- 模型容易欠拟合

### 4.3 太多会怎样

- 训练时间变长
- 可能过拟合
- 验证指标不再提升，甚至开始变差

### 4.4 怎么看要不要加 epochs

如果你看到：

- `train loss` 还在稳步下降
- `val` 指标也还在提升

那通常说明还没训够。

如果你看到：

- `train loss` 继续下降
- 但 `val loss` 或 `mAP` 不再变好

那说明可能已经接近过拟合了。

---

## 5. `batch_size` 是什么，怎么调

`batch_size` 表示：

**每次送进模型训练的样本数。**

### 5.1 API 例子

#### PyTorch 里

```python
train_loader = DataLoader(dataset, batch_size=16, shuffle=True)
```

#### Ultralytics 里

```python
model.train(data="data.yaml", batch=16)
```

### 5.2 batch 大小的直觉

- `batch` 大：一次看更多样本
- `batch` 小：一次看更少样本

### 5.3 batch 太大可能会怎样

- 显存占用高
- 容易 OOM

### 5.4 batch 太小可能会怎样

- 训练更慢
- 梯度噪声更大
- 收敛可能不稳定

### 5.5 现实里怎么调

最常见的做法不是“理论最优”，而是：

**在显存允许的前提下，尽量用一个合理偏大的 batch。**

如果你爆显存，第一优先级通常就是先减小 `batch`。

---

## 6. `learning rate` 是什么，怎么调

学习率 `lr` 表示：

**每次参数更新走多大一步。**

公式你已经见过：

```text
参数更新量 ≈ lr * grad
```

### 6.1 API 例子

#### PyTorch 里

```python
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)
```

#### Ultralytics 里

常见参数例如：

```python
model.train(data="data.yaml", lr0=0.01)
```

这里的 `lr0` 通常表示初始学习率。

### 6.2 学习率太大

常见表现：

- `loss` 大起大落
- 训练不稳定
- 指标乱跳
- 甚至 `loss = nan`

### 6.3 学习率太小

常见表现：

- `loss` 降得特别慢
- 训练很稳，但像没怎么学
- 很久都看不到明显提升

### 6.4 调学习率最实用的思路

如果你看到：

- 训练一开始就很不稳定

优先怀疑：

- `lr` 太大

如果你看到：

- 训练几乎不动，提升很慢

优先怀疑：

- `lr` 太小

---

## 7. `imgsz` 是什么，怎么调

`imgsz` 表示：

**输入模型前，图像被缩放到的尺寸。**

### 7.1 Ultralytics 例子

```python
model.train(data="data.yaml", imgsz=640)
```

### 7.2 imgsz 大的特点

- 图像细节更多
- 小目标更容易保留
- 但更吃显存、更慢

### 7.3 imgsz 小的特点

- 训练更快
- 占用更小
- 但小目标可能被缩得太厉害

### 7.4 什么情况下优先调 imgsz

如果你做的是：

- 小目标检测
- 远距离目标
- 密集场景

而且你发现：

- `Recall` 低
- 漏检多

那 `imgsz` 往往是值得优先考虑的参数之一。

但要注意：

**imgsz 往上调，通常显存和速度压力也会明显上升。**

---

## 8. `optimizer` 是什么，怎么选

优化器负责：

**根据梯度更新参数。**

### 8.1 PyTorch 常见 API

#### SGD

```python
optimizer = torch.optim.SGD(
    model.parameters(),
    lr=0.01,
    momentum=0.9
)
```

#### Adam

```python
optimizer = torch.optim.Adam(
    model.parameters(),
    lr=1e-3
)
```

#### AdamW

```python
optimizer = torch.optim.AdamW(
    model.parameters(),
    lr=1e-3,
    weight_decay=1e-4
)
```

### 8.2 直觉理解

- `SGD`：更经典、更朴素。每次更新走固定步长，需要手动调 lr schedule
- `Adam / AdamW`：自适应学习率。每个参数有自己的"专属步长"，通常更容易训起来

### 8.3 什么时候用哪个

| 场景 | 推荐 | 原因 |
|------|------|------|
| 刚入门、快速跑通 | Adam | 不用调太多参数就能收敛 |
| 小模型/小数据集 | Adam | 收敛快，性价比高 |
| 大模型/YOLO训练 | SGD | 很多经典检测模型用 SGD 训出来的，复现时对齐 |
| 追求最后1-2%的精度 | SGD + 精心调 lr | SGD 泛化能力在某些任务上略好 |

**关键区别**：Adam 内部会自动调整每个参数的学习率（所以对 lr 不敏感），SGD 对所有参数一视同仁（所以 lr 设错影响很大）。

Adam 的 lr 默认 0.001 基本都能跑，SGD 通常需要 lr=0.01~0.1 才跑得动。

### 8.4 你现在怎么记就够

入门阶段先记：

- 想快点稳稳训起来：常见会先试 `Adam` 或 `AdamW`
- 想和很多经典检测训练配置对齐：常见会看到 `SGD`

后面再慢慢补更细节差异。

---

## 9. `weight_decay` 是什么，怎么调

`weight_decay` 常被理解成：

**一种限制参数过大、帮助减轻过拟合的正则化手段。**

### 9.1 API 例子

```python
optimizer = torch.optim.AdamW(
    model.parameters(),
    lr=1e-3,
    weight_decay=1e-4
)
```

### 9.2 它常见在什么场景下有用

如果你发现：

- 训练集表现越来越好
- 验证集表现却跟不上

也就是出现过拟合迹象时，`weight_decay` 是一个常见可调项。

不过你现在先记：

**它不是第一优先级最先调的参数，但它是很常见的稳模型手段。**

---

## 10. `patience` 和早停 early stopping

很多训练过程里还会看到：

- `patience`

它表示：

**如果验证指标连续很多轮都没有提升，就提前停止训练。**

### 10.1 Ultralytics 例子

```python
model.train(data="data.yaml", epochs=100, patience=20)
```

意思是：

- 最多训练 100 轮
- 但如果验证效果连续 20 轮都没提升，就提早停掉

### 10.2 为什么它有用

它能帮你：

- 节省时间
- 减少无效训练
- 某种程度上防止后期过拟合继续恶化

## 11. 从训练现象反推该调什么

这一节是最实用的部分。

### 11.1 情况一：`loss` 不降

优先排查：

1. 学习率是不是太大或太小
2. 数据和标签有没有问题
3. 模型和损失函数是否匹配
4. 训练代码流程有没有写错

优先尝试：

- 降 `lr`
- 检查 batch 数据
- 先用小数据跑通

### 11.2 情况二：训练很慢

优先排查：

1. `batch_size` 是否太小
2. `imgsz` 是否太大
3. 数据加载是否太慢
4. 是否在用 GPU

优先尝试：

- 增大 `batch`
- 降低 `imgsz`
- 检查 `device`

### 11.3 情况三：过拟合

表现通常是：

- `train loss` 很漂亮
- `val loss` 或 `mAP` 不再提升

优先尝试：

- 增强数据
- 加大 `weight_decay`
- 减少 `epochs`
- 启用或收紧 `patience`

### 11.4 情况四：欠拟合

表现通常是：

- `train loss` 和 `val loss` 都不理想
- 模型整体学不会

优先尝试：

- 增加 `epochs`
- 检查 `lr`
- 用更强一点的模型
- 减弱不合理的增强

### 11.5 情况五：小目标漏检多

优先尝试：

- 提高 `imgsz`
- 检查是否需要更适合小目标的数据增强
- 回头看 `Recall`

### 11.6 情况六：误检太多

优先尝试：

- 分析 `Precision`
- 调整 `conf threshold`
- 增加负样本
- 优化数据质量

注意：

这里已经不只是训练超参数了，也会涉及推理阈值和数据问题。

## 11.7 NMS（非极大值抑制）— 容易被忽略的超参数

### 为什么一个物体会产生一堆重叠框？

YOLO 检测头在图像上密集地预测——8400 个位置，每个位置都可能输出框。同一个物体周围的好几个位置都会"看到"它，各自框一个：

```
同一只猫 → 3 个网格点抢着框：

┌─────────────┐
│  ┌─A──┐     │  ← A 框：置信 0.82
│  │  ┌─B──┐ │  ← B 框：置信 0.95（框得最准）
│  │  │猫脸│ │
│  │  └─C──┘ │  ← C 框：置信 0.71
│  └────────┘ │
└─────────────┘

最后一张图：几千个候选框 → 绝大多数是重叠的重复预测
```

### NMS 干什么

把多余的重叠框删掉，一个物体只留一个最好的。

```
1. 按置信度排序所有框
2. 取最高分框 A → 保留
3. 算其余框和 A 的 IoU（重叠面积/总面积）→ 超过阈值的 → 删
4. 在剩下的框里重复 2-3，直到处理完
```

### NMS IoU 阈值也是超参数

```python
# Ultralytics 里（预测时生效）
model.predict(source="image.jpg", iou=0.5)   # 默认 0.5

# 阈值低 (0.3)：删得狠 → 同一个目标只留 1 个框 ✅
#                → 但密集目标会被误杀 → 漏检 ❌
# 阈值高 (0.7)：删得轻 → 密集目标能共存 ✅
#                → 但同一个目标可能出多个框 → 误检 ❌
```

### 什么时候调 NMS

| 场景 | 调整方向 | 原因 |
|------|---------|------|
| 密集人群/停车位/货架商品 | **提高**阈值 (0.5→0.65) | 避免误杀相邻目标 |
| 同一个目标出多个重复框 | **降低**阈值 (0.5→0.3) | 去重更狠 |
| 默认 | 0.5 | 大多数场景合适 |

### 面试考点：密集人群漏检

```
问：两个人挨着，模型只检出一个人，为什么？

答：NMS 把第二个人当成了第一个人的重复框删掉了。
   因为两人 IoU > 0.5，过了默认阈值。

解决：提高 NMS IoU 阈值，或用 Soft-NMS（不直接删，降置信度）。
```

> 可视化演示：跑 `python 00_basics/10_nms_demo.py`，直观对比不同 IoU 阈值的抑制效果。

## 12. 训练指标和调参的关系

调参不能脱离指标。

你要学会把：

- `train loss`
- `val loss`
- `Precision`
- `Recall`
- `mAP`

一起看。




### 12.1 只看 train loss 不够

因为模型可能只是把训练集记住了。

### 12.2 只看 mAP 也不够

你还要看：

- 是误检多还是漏检多
- 是框不准还是类混淆

### 12.3 最实用的习惯

每次调参后都问自己：

1. `train loss` 变了没有
2. `val loss` 变了没有
3. `Precision / Recall` 哪个变了
4. `mAP50` 和 `mAP50-95` 是否一起变化

## 13. 一个 PyTorch 训练参数示例

下面是一个更贴近你现在理解层级的例子：

```python
import torch
import torch.nn as nn
from torch.utils.data import DataLoader

model = MyModel().to(device)
train_loader = DataLoader(train_dataset, batch_size=16, shuffle=True)
val_loader = DataLoader(val_dataset, batch_size=16, shuffle=False)

loss_fn = nn.CrossEntropyLoss()
optimizer = torch.optim.AdamW(
    model.parameters(),
    lr=1e-3,
    weight_decay=1e-4
)

num_epochs = 30

for epoch in range(num_epochs):
    model.train()

    for x_batch, y_batch in train_loader:
        x_batch = x_batch.to(device)
        y_batch = y_batch.to(device)

        optimizer.zero_grad()
        pred = model(x_batch)
        loss = loss_fn(pred, y_batch)
        loss.backward()
        optimizer.step()
```

这里可以调的典型超参数就是：

- `batch_size=16`
- `lr=1e-3`
- `weight_decay=1e-4`
- `num_epochs=30`

## 15. 一个更高效的调参顺序

如果你现在问：

**到底先调哪个，后调哪个？**

我建议你先按这个顺序：

### 第一步：先跑 baseline

先不要乱动太多参数。

先得到一个基础结果。

### 第二步：先解决“能不能正常训”

优先看：

- loss 是否正常下降
- 有没有爆显存
- 有没有发散

这一步主要动：

- `batch`
- `lr`
- `imgsz`

### 第三步：再看“效果好不好”

优先看：

- `Precision`
- `Recall`
- `mAP`

这一步可能动：

- 增强参数
- `epochs`
- `imgsz`
- 推理阈值

### 第四步：再做细调

如果已经能跑且效果还行，再考虑：

- `optimizer`
- `weight_decay`
- `patience`

### 一句话版顺序

**先保训练稳定，再追效果提升，最后做细节优化。**

## 16. 不要这样调参

这是很重要的一节。

### 16.1 不要一次改很多参数

否则你根本不知道是哪一个起作用。

### 16.2 不要只凭感觉调

一定要结合：

- 曲线
- 验证指标
- 可视化结果

### 16.3 不要只盯一个指标

比如：

- `Precision` 高，不一定整体就好
- `mAP50` 高，不一定 `mAP50-95` 好

### 16.4 不要把数据问题误当成参数问题

如果标注很乱，调半天参数也救不回来。

## 17. 这一节最该记住的几句话

### 17.1 调参的定义

**调参 = 根据训练现象和验证结果，有目标地调整超参数。**

### 17.2 参数和超参数的区别

**模型参数是学出来的，超参数是你定的。**

### 17.3 最关键的几个超参数

**先盯住 `epochs / batch_size / lr / imgsz / optimizer / weight_decay`。**

### 17.4 调参顺序

**先保证训练稳定，再追求效果提升。**

### 17.5 调参方法

**一次少改几个，并且始终结合 `loss + 指标 + 可视化` 一起看。**

## 18. 一张调参思维图

```text
先跑 baseline
-> 看 train/val loss 是否正常
-> 看 Precision / Recall / mAP 哪个掉链子
-> 判断是训练不稳定、过拟合、欠拟合、漏检还是误检
-> 对应去调 lr / batch / imgsz / epochs / 增强 / 优化器 等超参数
-> 每次少量修改
-> 再次验证
```

## 20. 实战案例：调参日记

下面是一份真实的调参记录，用的就是你写的 CNN（CIFAR-10，目标 60%+）：

### 第 0 轮：Baseline

```python
lr=0.001, batch_size=64, epochs=10, optimizer=Adam, weight_decay=0
```

结果：测试准确率 69.7%，train loss 从 1.44 降到 0.44

观察：loss 稳定下降，没有震荡，说明 lr 合适。train loss 还在降，可能还没训够。

### 第 1 轮：加 epochs

```python
lr=0.001, batch_size=64, epochs=20, optimizer=Adam, weight_decay=0  ← 只改 epochs
```

预期：准确率可能继续提升到 72-73%
如果 train loss 还在降但 test acc 不再提升 → 过拟合，下一个该加 weight_decay

### 第 2 轮：如果过拟合，加 weight_decay

```python
lr=0.001, batch_size=64, epochs=20, optimizer=Adam, weight_decay=1e-4  ← 加正则化
```

### 第 3 轮：换 SGD 试试

```python
lr=0.01, batch_size=64, epochs=20, optimizer=SGD, momentum=0.9  ← 注意 lr 要大！
```

SGD 用 lr=0.001 基本走不动，必须用 0.01 或更大。

### 调参铁律

```
一次只改一个参数
改之前记录当前结果
改之后对比
```

这样你才知道哪个改动真正起了作用。
