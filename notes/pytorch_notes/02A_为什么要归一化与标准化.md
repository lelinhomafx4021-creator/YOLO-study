# 为什么要归一化与标准化

## 1. 这节解决什么问题

很多人第一次看到这段代码都会疑惑：

```python
transform = transforms.Compose([
    transforms.ToTensor(),
    transforms.Normalize((0.5, 0.5, 0.5), (0.5, 0.5, 0.5))
])
```

疑问通常有 4 个：

1. 图片本来好好的，为什么非要改数值范围。
2. `ToTensor()` 和 `Normalize()` 到底谁在做什么。
3. `mean` 和 `std` 是什么，为什么公式是 `(x - mean) / std`。
4. 这件事为什么会影响训练速度、梯度稳定性和最终效果。

这节专门把这些问题讲透。

---

## 2. 先说结论

这段预处理做了两件事：

1. `ToTensor()`：把图片从常见的 `0~255` 像素值转成 `0~1` 的浮点数，并把维度从 `H x W x C` 变成 `C x H x W`。
2. `Normalize(mean, std)`：再按公式 `x' = (x - mean) / std` 调整数据分布。

如果 `mean=0.5`，`std=0.5`，那么 `0~1` 会进一步映射到大致 `-1~1`。

---

## 3. 先分清两个词

### 3.1 入门场景里大家常把它们混着叫

很多教程会把下面两步都叫“归一化”：

1. `x / 255`
2. `(x - mean) / std`

但严格一点说，它们不是同一件事。

### 3.2 更严格的区分

`x / 255`

- 更像范围缩放
- 把数值从 `0~255` 缩到 `0~1`

`(x - mean) / std`

- 更像标准化
- 先减均值，再除标准差

所以你学习时最好这样记：

1. `ToTensor()` 负责把图片转 tensor，并把像素缩放到 `0~1`
2. `Normalize()` 负责按均值和标准差继续调整分布

---

## 4. `ToTensor()` 到底在做什么

原始图片像素通常是整数，例如：

```text
0, 1, 2, ..., 255
```

如果某个像素值是 `128`，经过 `ToTensor()` 后会变成：

```text
128 / 255 = 0.50196
```

所以：

```text
0   -> 0.0
128 -> 0.502
255 -> 1.0
```

数学上就是：

```text
x' = x / 255
```

这一步的作用很直接：

1. 把输入数值压到更小、更统一的范围。
2. 让后面模型计算时，数值不要一上来就很大。

---

## 5. `Normalize(mean, std)` 到底在做什么

它的公式是：

```text
x' = (x - mean) / std
```

这个公式做了两件事：

1. `x - mean`：把数据中心挪到 `0` 附近。
2. `/ std`：把不同特征的波动尺度拉到更统一。

### 5.1 `mean` 是什么

`mean` 就是平均值。

例如一组数：

```text
0.2, 0.4, 0.6, 0.8
```

平均值是：

```text
mean = (0.2 + 0.4 + 0.6 + 0.8) / 4 = 0.5
```

如果只做 `x - mean`：

```text
0.2 -> -0.3
0.4 -> -0.1
0.6 ->  0.1
0.8 ->  0.3
```

现在这些数就不再整体偏正，而是围着 `0` 分布。

这一步叫中心化，或者去均值。

### 5.2 `std` 是什么

`std` 是标准差，表示“数据离平均值散得有多开”。

还是这组数：

```text
0.2, 0.4, 0.6, 0.8
mean = 0.5
```

每个数减去均值：

```text
-0.3, -0.1, 0.1, 0.3
```

平方：

```text
0.09, 0.01, 0.01, 0.09
```

平均：

```text
(0.09 + 0.01 + 0.01 + 0.09) / 4 = 0.05
```

再开根号：

```text
std = sqrt(0.05) ≈ 0.2236
```

所以标准化后：

```text
(0.2 - 0.5) / 0.2236 ≈ -1.34
(0.4 - 0.5) / 0.2236 ≈ -0.45
(0.6 - 0.5) / 0.2236 ≈  0.45
(0.8 - 0.5) / 0.2236 ≈  1.34
```

这时数据不但围绕 `0`，而且尺度也统一了。

---

## 6. 为什么非要这么做

核心原因不是“格式要求”，而是“训练更稳定”。

### 6.1 原始输入数值太大，不利于训练

如果直接喂 `0~255`，那么第一层算出来的值也容易偏大。

例如最简单的线性层：

```text
y = wx + b
```

如果 `x` 动不动就是 200、240、255，那么 `wx` 的数值也容易很大。

这会带来几个问题：

1. 每层激活值尺度更难控制。
2. 学习率更难调。
3. 梯度更容易忽大忽小。

### 6.2 输入围绕 0，优化通常更自然

如果输入大多是正数，例如都在 `0~1`，模型前面几层看到的输入往往整体偏正。

如果经过中心化后数据围绕 `0`：

```text
-0.8, -0.2, 0.1, 0.7
```

模型看到的输入会更平衡。

这通常会让训练更顺。

### 6.3 各通道、各特征尺度统一，更公平

如果一个特征范围很小，另一个特征范围很大，那么大范围特征会更容易主导训练。

标准化后，不同特征会被拉回接近的尺度。

这样优化器更新参数时，不会特别偏向某一个大尺度输入。

---

## 7. 它和梯度有什么关系

这是最重要的地方之一。

看一个最简单的例子：

```text
y = wx + b
L = (y - t)^2
```

对 `w` 求导：

```text
∂L/∂w = 2(y - t)x
```

这里最关键的是最后那个 `x`。

也就是说：

1. 如果 `x` 很大，梯度往往也会更大。
2. 如果 `x` 的分布很乱，梯度尺度也容易乱。
3. 如果 `x` 被处理到比较稳定的范围，梯度通常也更稳定。

所以归一化/标准化不是“为了规范”，而是会直接影响参数更新过程。

---

## 8. 放到图像里看一遍

一张 RGB 图片，某个像素假设是：

```text
R = 255, G = 128, B = 0
```

### 8.1 先经过 `ToTensor()`

```text
R = 255 / 255 = 1.0
G = 128 / 255 ≈ 0.502
B = 0 / 255 = 0.0
```

### 8.2 再经过 `Normalize((0.5, 0.5, 0.5), (0.5, 0.5, 0.5))`

每个通道都做：

```text
x' = (x - 0.5) / 0.5
```

所以：

```text
R -> (1.0 - 0.5) / 0.5 = 1.0
G -> (0.502 - 0.5) / 0.5 ≈ 0.004
B -> (0.0 - 0.5) / 0.5 = -1.0
```

最后这个像素就大致变成：

```text
[1.0, 0.004, -1.0]
```

---

## 9. 为什么 `(0.5, 0.5, 0.5)` 会把 `0~1` 变成 `-1~1`

因为：

```text
x' = (x - 0.5) / 0.5
```

代入边界值：

```text
0.0 -> (0.0 - 0.5) / 0.5 = -1
0.5 -> (0.5 - 0.5) / 0.5 = 0
1.0 -> (1.0 - 0.5) / 0.5 = 1
```

所以这一组参数非常好记：

1. 先把数据中心放到 `0`
2. 再把范围大致映射到 `-1~1`

---

## 10. 为什么是 3 个数

你会看到：

```python
Normalize((0.5, 0.5, 0.5), (0.5, 0.5, 0.5))
```

这里有 3 个 `mean`，3 个 `std`，因为彩色图像有 3 个通道：

1. 红色通道 `R`
2. 绿色通道 `G`
3. 蓝色通道 `B`

也就是说它不是整张图只算一次，而是：

```text
R' = (R - mean_R) / std_R
G' = (G - mean_G) / std_G
B' = (B - mean_B) / std_B
```

这叫按通道标准化。

---

## 11. 为什么有时候不用 `0.5`

入门教程常写：

```python
Normalize((0.5, 0.5, 0.5), (0.5, 0.5, 0.5))
```

因为它简单、直观、好理解。

但更严谨的做法是用数据集真实统计值，例如 CIFAR10 常见写法：

```python
mean = (0.4914, 0.4822, 0.4465)
std = (0.2023, 0.1994, 0.2010)
```

意思是：

1. 这是从整个 CIFAR10 训练集统计出来的 RGB 均值。
2. 这是对应的 RGB 标准差。
3. 这样处理后，更贴近这个数据集自己的真实分布。

入门时先理解 `0.5 -> -1~1` 的逻辑就够了。

## 12. 它在你当前代码里到底起什么作用

你现在做 CIFAR10 分类时：

```python
transform = transforms.Compose([
    transforms.ToTensor(),
    transforms.Normalize((0.5, 0.5, 0.5), (0.5, 0.5, 0.5))
])
```

可以直接理解成：

1. 先把像素从 `0~255` 压到 `0~1`
2. 再把 `0~1` 平移缩放到 `-1~1`
3. 最后把这个更稳定的输入送进卷积网络

这样做的目标不是“让图更好看”，而是“让模型更容易学”。

---

## 13. 最容易混淆的 6 个点

### 13.1 `ToTensor()` 不只是转类型

它不只是把图片变成 tensor，还会把很多常见图像从 `[0,255]` 缩放到 `[0,1]`。

### 13.2 `Normalize()` 不是“自动算 mean/std`

你传进去多少，它就按多少算。

例如：

```python
Normalize((0.5, 0.5, 0.5), (0.5, 0.5, 0.5))
```

这里的 `0.5` 不是程序自己算出来的，是你手动指定的。

### 13.3 `Normalize()` 不是只减均值

它是：

```text
先减 mean，再除 std
```

两步缺一不可。

### 13.4 训练和预测必须用同样的预处理

训练时如果用了：

```python
ToTensor() + Normalize(...)
```

预测时也必须一致。

不一致会导致输入分布变掉，结果很容易飘。

### 13.5 分类图像和 mask 不能乱用同样的 `ToTensor()`

像语义分割里的 mask 不是普通图像强度，不应该随便按图像方式缩放到 `0~1`。

### 13.6 这一步不是为了“提高图片质量”

它是为了调整输入分布，优化训练过程。

---

## 14. 一个最小可跑例子

```python
import torch
import torchvision
import torchvision.transforms as transforms

transform = transforms.Compose([
    transforms.ToTensor(),
    transforms.Normalize((0.5, 0.5, 0.5), (0.5, 0.5, 0.5)),
])

dataset = torchvision.datasets.CIFAR10(
    root="./data",
    train=True,
    download=False,
    transform=transform
)

img, label = dataset[0]

print(img.shape)       # torch.Size([3, 32, 32])
print(img.dtype)       # torch.float32
print(img.min().item())
print(img.max().item())
print(label)
```

你会看到：

1. 图像已经是 tensor。
2. 形状从 `HWC` 变成了 `CHW`。
3. 数值范围大致落在 `-1~1` 左右。

---

## 15. 这节最该记住的 8 句话

1. 原始图像像素通常是 `0~255` 的整数。
2. `ToTensor()` 会把它变成 `0~1` 的浮点数，并调整维度顺序。
3. `Normalize(mean, std)` 的公式是 `x' = (x - mean) / std`。
4. `减 mean` 是为了让数据围绕 `0`。
5. `除 std` 是为了让不同特征的尺度更统一。
6. 输入分布更稳定，梯度通常也更稳定。
7. 这会让模型更容易训练、学习率更容易调。
8. 训练和预测必须保持同样的预处理流程。
