# 自注意力机制详解：Q、K、V 到底是什么

## 这一节在学习顺序里的位置

上一节你知道了：

- Transformer 用注意力机制，让每个词都能看到所有词
- 不像 RNN 一个一个读，可以并行

但"注意力"到底是怎么算的？Q、K、V 又是什么？

这一节把计算过程拆开，一步一步讲清楚。

**配合可视化看效果更好：[自注意力计算过程可视化](self_attention_visual.html)**

---

## 先说最核心的一句话

**Q = 我在找什么，K = 我能提供什么，V = 我的实际内容。Q 和 K 匹配出权重，权重决定取多少 V。**

---

## 1. 为什么需要 Q、K、V

假设你有一个句子 "我 今天 去 北京"，每个词用一个向量表示。

你想让每个词知道"我应该关注哪些词"。怎么做？

**直觉：每个词发出两种信号——"我在找什么"（Q）和"我能提供什么"（K）。**

```
"我" 的 Q: 我在找一个地名
"北京" 的 K: 我是一个地名

Q 和 K 匹配上了 → "我" 应该关注 "北京"
```

V 是另一个东西：**我的实际内容是什么**。匹配上之后，就用 V 的内容。

---

## 2. Q、K、V 怎么来的

每个词的原始向量 × 三个不同的权重矩阵 → 得到 Q、K、V。

```
"我" 的向量 = [1.0, 0.5, 0.2, 0.8]

Q_我 = 向量 × W_Q = [1.5, 1.3, 1.2, 0.6]
K_我 = 向量 × W_K = [0.7, 1.5, 1.0, 1.1]
V_我 = 向量 × W_V = [1.5, 1.5, 0.2, 0.5]
```

**三个权重矩阵 W_Q、W_K、W_V 是训练时学的。** 不同的矩阵让 Q、K、V 关注不同的信息。

---

## 3. 怎么算"该关注谁"

拿 "我" 的 Q 和所有词的 K 做点积：

```
"我" 的 Q:    [1.5, 1.3, 1.2, 0.6]
"我" 的 K:    [0.7, 1.5, 1.0, 1.1]  → 点积 = 1.5×0.7 + 1.3×1.5 + 1.2×1.0 + 0.6×1.1 = 4.80
"今天" 的 K:  [1.0, 0.1, 0.6, 0.7]  → 点积 = 1.5×1.0 + 1.3×0.1 + 1.2×0.6 + 0.6×0.7 = 3.09
"去" 的 K:    [0.4, 1.0, 1.0, 0.7]  → 点积 = 1.5×0.4 + 1.3×1.0 + 1.2×1.0 + 0.6×0.7 = 3.92
"北京" 的 K:  [0.2, 0.9, 1.5, 1.0]  → 点积 = 1.5×0.2 + 1.3×0.9 + 1.2×1.5 + 0.6×1.0 = 3.87
```

**点积越大 = 两个向量越"对齐" = 关系越强。**

---

## 4. 为什么要除以 √d_k

上面的点积值可能很大（4.80, 3.09...）。直接扔进 Softmax 会有问题：

```
如果分数是 [4.8, 3.1, 3.9, 3.9]:
  Softmax → [0.45, 0.13, 0.21, 0.21]  ← 分布还行

如果分数是 [48, 31, 39, 39]:
  Softmax → [0.99, 0.00, 0.01, 0.00]  ← 几乎全给第一个了！
```

**分数太大会导致 Softmax 输出接近 one-hot，梯度消失，训练不动。**

除以 √d_k（d_k 是向量维度）把分数压到合理范围：

```
分数 / √4 = 分数 / 2
[4.8, 3.1, 3.9, 3.9] / 2 = [2.4, 1.55, 1.95, 1.95]
Softmax → [0.37, 0.16, 0.24, 0.24]  ← 分布更均匀
```

**这一步叫 "Scaled Dot-Product Attention"，"Scaled" 就是除以 √d_k。**

---

## 5. Softmax：把分数变成概率

Softmax 做两件事：

1. **归一化**：每行加起来 = 1
2. **放大差距**：大的更大，小的更小

```
原始分数:    [2.40, 1.55, 1.95, 1.95]
Softmax后:  [0.37, 0.16, 0.24, 0.24]  ← 加起来 = 1.0
```

**现在每个词都有一个"关注度"分布：该关注谁、关注多少。**

---

## 6. 加权求和 V：得到新表示

最后一步：用 Softmax 的权重对 V 做加权求和。

```
"我" 的输出 = 0.37 × V_我 + 0.16 × V_今天 + 0.24 × V_去 + 0.24 × V_北京
```

**"我" 的新向量 = "我"的 V 占 37% + "今天"的 V 占 16% + "去"的 V 占 24% + "北京"的 V 占 24%。**

这就是"注意力"的含义：每个词的新表示，是所有词的内容按关注度混合的结果。

---

## 7. 为什么叫"自"注意力

"自" = 自己和自己。

**输入序列和输出序列是同一组词。** "我"看"今天""去""北京"——都是同一句话里的词。

对比"交叉注意力"（Cross-Attention）：

```
自注意力:  Q 来自序列A，K/V 也来自序列A → 自己关注自己
交叉注意力: Q 来自序列A，K/V 来自序列B → A 去关注 B
```

交叉注意力用在 DETR（检测器的 Transformer 关注图片特征）和机器翻译（解码器关注编码器）。

---

## 8. 完整公式

```
Attention(Q, K, V) = Softmax(Q · K^T / √d_k) · V
```

就这一行。所有复杂的描述最后都归结到这个公式。

---

## 9. 这一节最该记住的 5 句话

1. **Q = 我在找什么，K = 我能提供什么，V = 我的实际内容**
2. **Q 和 K 的点积越大 = 两个词关系越强**
3. **除以 √d_k 防止分数太大导致梯度消失**
4. **Softmax 把分数变成概率，加起来 = 1**
5. **输出 = 所有词的 V 按注意力权重加权混合**

---

## 10. 学完这一节后你应该能回答的问题

**Q1: Q、K、V 分别代表什么**

A: **Q = 我在找什么（Query），K = 我能提供什么（Key），V = 我的实际内容（Value）。**

"我"的 Q 是"我在找一个地名"，"北京"的 K 是"我是一个地名"。Q 和 K 匹配上了，就取"北京"的 V 混入"我"的输出。

---

**Q2: Q 和 K 的点积为什么能衡量两个词的关系**

A: **点积 = 两个向量的"对齐程度"。**

方向一致 → 点积大 → 关系强。方向垂直 → 点积 ≈ 0 → 没关系。方向相反 → 点积负 → 负相关。

---

**Q3: 为什么要除以 √d_k**

A: **防止分数太大导致 Softmax 输出接近 one-hot，梯度消失。**

维度越大，点积值越大。分数太大 → Softmax 输出 [0.99, 0.00, 0.01, 0.00] → 梯度几乎为零 → 训练不动。除以 √d_k 把分数压到合理范围。

---

**Q4: Softmax 在这里起什么作用**

A: **把分数变成概率（加起来 = 1），同时放大差距。**

Softmax 让大的更大、小的更小。高分词的权重被放大，低分词被压到接近 0。注意力集中在最相关的几个词上。

---

**Q5: 最终的输出是怎么算出来的**

A: **用 Softmax 的权重对所有词的 V 做加权求和。**

输出 = 权重1 × V1 + 权重2 × V2 + 权重3 × V3 + ...。权重越大的词，它的 V 对输出的影响越大。

---

**Q6: "自注意力"和"交叉注意力"的区别是什么**

A: **自注意力 = Q/K/V 都来自同一序列（自己关注自己）。交叉注意力 = Q 来自一个序列，K/V 来自另一个序列。**

自注意力用在 Encoder（一句话里的词互相看）。交叉注意力用在 DETR（Query 去看图片特征）和机器翻译（解码器关注编码器）。

---

**Q7: 完整公式是什么**

A: **Attention(Q, K, V) = Softmax(Q · K^T / √d_k) · V**

就这一行。所有复杂的描述最后都归结到这个公式。

---

## 复习速答

- `Q`：我在找什么。
- `K`：我能提供什么索引。
- `V`：真正要取走的内容。
- `√d_k`：防止分数过大。
- `Attention`：先算权重，再对 V 加权求和。
