DETR Object Query 可视化

Object Query 通过交叉注意力去图片里找物体

场景：一张图里有 3 个物体

图里有：猫（左上）、球（右下）、骨头（中间）。
DETR 预设了 5 个 Object Query。每个 Query 负责找一个物体。

Step 1: Object Query 是什么

Query

Object Query 是 5 个可学习的向量，每个 Query 带一个"初始位置偏好"。
训练后，Query 1 可能学会"我负责找左上区域的物体"，Query 2 学会"我负责找右下区域"...

点击看每个 Query 的位置偏好。训练前是随机的，训练后会分化。

Step 2: 交叉注意力 — Query 去图片里找

交叉注意力

Query 的 Q 去和图片特征的 K 做点积 → 算出"这个 Query 应该关注图片的哪些区域"。
然后用注意力权重对图片特征的 V 做加权求和 → Query 拿到了它负责区域的信息。

Query输出 = Softmax(Q_query · K_imageT / √d) · V_image

Step 3: Query 之间互相看（自注意力）

自注意力

5 个 Query 之间做自注意力。
Query 1 和 Query 2 都关注了左上区域 → 自注意力后它们知道"对方也在找左上" → 避免重复。

自注意力让不同的 Query 负责不同的物体，不会都去抢同一个。

Step 4: 输出预测

输出

每个 Query 经过 FFN 后输出一个预测：(框坐标, 类别)。
训练时用匈牙利算法把 Query 和真实物体配对。没配上的 Query 输出 "no object"。

和 YOLO 的对比

YOLO: 每个格子预测 N 个框 → 几千个候选 → NMS 去重

DETR: 5 个 Query 各找一个物体 → 5 个直接结果 → 不需要 NMS

DETR = 端到端，直接出结果。YOLO = 多步，需要后处理。
