# 评估指标详解

## 这一节为什么重要

训练跑完之后，最自然的问题就是：

**模型到底好不好？**

这件事不能只靠肉眼看几张图，也不能只看“loss 有没有降”。

因为：

- `loss` 是训练优化的目标
- 指标是我们评价模型效果的工具

所以这一节的任务就是把常见指标真正讲清楚，尤其是你现在做目标检测最常用的这些：

- `IoU`
- `Precision`
- `Recall`
- `F1`
- `AP`
- `mAP50`
- `mAP50-95`

这几个指标一旦理顺，后面看 `val` 输出就不再只是看热闹。

---

## 先说结论：指标在回答不同问题

这些指标虽然都在评价模型，但它们关注点不一样：

- `IoU`：框画得准不准
- `Precision`：你报出来的结果里，有多少是真的
- `Recall`：真实目标里，你抓到了多少
- `F1`：精确率和召回率是否平衡
- `AP / mAP`：综合表现怎么样

所以不要把它们理解成“很多个差不多的分数”，它们各自回答的是不同问题。

---

## 1. 先把 TP / FP / FN 讲清楚

学检测指标前，必须先理解三个基础记号：

- `TP`：True Positive，真正例
- `FP`：False Positive，假正例
- `FN`：False Negative，假负例

在目标检测里，它们可以这样理解：

### 1.1 TP

模型预测了一个框，

- 类别对了
- 框的位置也够准
- 通常还满足某个 `IoU` 阈值

那这个预测就算 `TP`。

### 1.2 FP

模型预测了一个框，但：

- 这个框本来就不该有
- 或者类别错了
- 或者框和真实框重叠不够

那就算 `FP`。

### 1.3 FN

真实目标明明存在，但模型没有成功检测到，

那就是 `FN`。

一句话记忆：

- `TP`：报对了
- `FP`：报错了
- `FN`：漏掉了

---

## 2. IoU 是什么

`IoU` 全称是：

**Intersection over Union**

中文通常叫：

**交并比**

它的作用是：

**衡量预测框和真实框重叠得有多好。**

公式：

```text
IoU = 交集面积 / 并集面积
```

### 2.1 IoU 取值范围

- `0`：完全不重叠
- `1`：完全重合

值越大，说明框的位置越准。

### 2.2 为什么 IoU 很重要

因为在目标检测里，不是“只要画了一个框就算对”，还要看框的位置是否足够接近真实框。

所以很多指标都会先问：

**这个预测框和真实框的 IoU 有没有达到阈值？**

最常见阈值之一是：

```text
IoU >= 0.50
```

这时才可能把这个预测记成一个 `TP`。

### 2.3 IoU 只回答定位问题

这一点很重要。

IoU 高，只能说明：

- 框位置画得不错

它不能单独说明：

- 类别一定对
- 模型整体一定好

所以 IoU 是定位指标，不是总评分。

---

## 3. Precision 是什么

`Precision` 中文常叫：

**精确率**

它的公式是：

```text
Precision = TP / (TP + FP)
```

它回答的问题是：

**模型报出来的“正例”里，有多少是真的。**

### 3.1 检测任务里的直觉

比如模型说：

- “我检测到了 10 个安全帽”

结果里面：

- 8 个是真的安全帽
- 2 个其实是背景误检

那：

```text
Precision = 8 / 10 = 0.8
```

### 3.2 Precision 高说明什么

说明：

**误检少**

也就是模型一旦报目标，往往比较靠谱。

### 3.3 Precision 低常见意味着什么

说明：

**误检多**

也就是模型把很多本不该报的东西也报出来了。

---

## 4. Recall 是什么

`Recall` 中文常叫：

**召回率**

它的公式是：

```text
Recall = TP / (TP + FN)
```

它回答的问题是：

**真实存在的目标里，模型抓到了多少。**

### 4.1 检测任务里的直觉

比如图片里真实有：

- 10 个安全帽

模型成功检测到了：

- 7 个

漏了：

- 3 个

那：

```text
Recall = 7 / 10 = 0.7
```

### 4.2 Recall 高说明什么

说明：

**漏检少**

### 4.3 Recall 低常见意味着什么

说明：

**漏检多**

比如：

- 小目标没抓到
- 遮挡目标没抓到
- 远距离目标没抓到

---

## 5. Precision 和 Recall 为什么经常打架

这是最该理解的地方之一。

很多时候：

- 你想提高 `Precision`
- 往往会让 `Recall` 降一点

反过来也一样。

### 5.1 为什么会这样

因为模型输出通常带置信度分数。

如果你把置信度阈值调高：

- 很多低分预测被过滤掉
- `FP` 往往会减少
- `Precision` 往往上升

但同时：

- 一些原本就是真的目标，也可能被你过滤掉
- `FN` 增加
- `Recall` 下降

### 5.2 所以不能只看一个指标

如果你只追求高 `Precision`，模型可能会：

- 只在非常确定时才报目标
- 结果非常保守
- 漏掉很多真实目标

如果你只追求高 `Recall`，模型可能会：

- 什么都想报一下
- 不容易漏
- 但误检会很多

所以真正好的模型通常是：

**Precision 和 Recall 都比较平衡。**

---

## 6. F1 Score 是什么

`F1` 是：

**Precision 和 Recall 的调和平均。**

公式：

```text
F1 = 2PR / (P + R)
```

它的作用是：

**用一个数衡量 Precision 和 Recall 的平衡情况。**

### 6.1 为什么不用普通平均

因为调和平均对“某一项特别低”的情况更敏感。

例如：

- Precision 很高，但 Recall 很低

这时 `F1` 不会给你太好看的分数。

### 6.2 F1 什么时候有用

当你不想只盯误检，也不想只盯漏检时，`F1` 是个很好的平衡指标。

---

## 7. AP 是什么

`AP` 全称：

**Average Precision**

它不是某一个固定阈值下的单点分数，而是：

**Precision-Recall 曲线下面积。**

### 7.1 为什么会有 Precision-Recall 曲线

因为当你调整置信度阈值时：

- Precision 会变
- Recall 也会变

所以模型不是只有一个 `(P, R)` 点，而是会形成一条曲线。

`AP` 就是在总结这条曲线整体表现。

### 7.2 AP 的直觉理解

你可以先这样理解：

**AP 越高，说明这个类别在各种阈值下整体表现越好。**

---

## 8. mAP 是什么

`mAP` 全称：

**Mean Average Precision**

意思是：

**把多个类别的 AP 求平均。**

例如有三个类别：

- `helmet`
- `no-helmet`
- `safety-vest`

那：

```text
mAP = (AP_helmet + AP_no-helmet + AP_safety-vest) / 3
```

### 8.1 为什么 mAP 常被当总指标

因为它把：

- 多类别表现
- Precision / Recall 曲线整体表现

综合进了一个分数里。

所以在目标检测里，`mAP` 是最常被拿来做总体评价的指标之一。

---

## 9. mAP50 和 mAP50-95 的区别

这是目标检测里特别常见的一对指标。

### 9.1 mAP50

意思是：

**在 IoU 阈值 = 0.50 时计算出来的 mAP。**

也就是：

- 只要框和真值框重叠超过 50%
- 就算满足匹配条件

它相对宽松一些。

### 9.2 mAP50-95

意思是：

**在多个 IoU 阈值上计算 mAP，再求平均。**

通常是：

```text
0.50, 0.55, 0.60, ..., 0.95
```

所以它比 `mAP50` 更严格。

### 9.3 为什么 mAP50-95 更难看

因为它要求：

- 不只是“差不多框住”
- 而是要在更严格的重叠标准下也表现好

所以通常你会看到：

- `mAP50` 比较高
- `mAP50-95` 明显更低

这是正常现象。

---

## 10. 在 Ultralytics 的 `val` 输出里怎么读

你常会看到类似：

```text
Box(P, R, mAP50, mAP50-95)
```

这四个值分别表示：

- `P`：Precision
- `R`：Recall
- `mAP50`
- `mAP50-95`

它们会：

- 给一个总平均
- 也可能给每个类别单独的结果

### 10.1 类别级别指标有什么意义

如果总分看起来还行，但某一类很差，类别级指标能帮你定位问题。

比如：

- `helmet` 很好
- `no-helmet` 很差

那你就知道不是模型全坏了，而是这一个类需要重点分析。

---

## 11. 指标和阈值的关系

这是实战里很容易忽略的一点。

模型预测时通常有两个非常重要的阈值：

- `confidence threshold`
- `IoU threshold`

### 11.1 confidence threshold

它会影响：

- 多少预测被保留

阈值高：

- 更保守
- Precision 常上升
- Recall 常下降

阈值低：

- 更激进
- Recall 常上升
- Precision 常下降

### 11.2 IoU threshold

它会影响：

- 多严格才算“这个框是对的”

阈值高：

- 框要画得更准才算对

阈值低：

- 框稍微重叠就可能算对

所以看指标时，一定要知道：

**指标不是脱离阈值独立存在的。**

---

## 12. 指标低了怎么诊断

这一步最有实战价值。

### 12.1 Precision 低

常见解释：

- 误检多
- 背景中很多东西被认成目标
- 置信度阈值太低

常见方向：

- 提高推理置信度阈值
- 增加负样本
- 增加更容易混淆背景的数据

### 12.2 Recall 低

常见解释：

- 漏检多
- 小目标没识别到
- 遮挡、远距离、模糊目标抓不到

常见方向：

- 增加数据
- 增强特征表达
- 提高输入分辨率
- 针对弱类补数据

### 12.3 IoU 低

常见解释：

- 模型能找到目标，但框位置不够准

常见方向：

- 检查标注质量
- 提升定位能力
- 检查输入缩放与标注对齐

### 12.4 mAP 低

常见解释：

- 说明整体表现还有明显短板
- 需要回到类别级指标和坏样例继续拆

---

## 13. 为什么不能只看 loss

这是很多初学者容易踩的坑。

训练时你会看到：

- `train loss`
- `val loss`

但检测任务最后更常关心的是：

- `Precision`
- `Recall`
- `mAP`

因为：

- loss 是训练优化时用的内部目标
- 指标更接近你最终业务关心的效果

比如：

- loss 下降了
- 但误检还是很多

那业务上仍然不一定满意。

所以：

**loss 用来判断训练过程，指标用来判断模型效果。**

---

## 14. 为什么还要看曲线和混淆矩阵

光看单个数字还不够。

Ultralytics 通常会给你：

- `PR_curve`
- `P_curve`
- `R_curve`
- `F1_curve`
- `confusion_matrix`

### 14.1 PR 曲线

看的是：

**Precision 和 Recall 在不同阈值下怎么权衡。**

### 14.2 F1 曲线

看的是：

**在哪些阈值附近，Precision 和 Recall 平衡得更好。**

### 14.3 混淆矩阵

看的是：

**模型最容易把什么类认成什么类。**

这在多类别任务里非常有用。

所以真正分析模型，不是只看一个 `mAP` 数字，而是：

**数字 + 曲线 + 可视化 + 坏样例**

一起看。

---

## 15. 一个直觉例子：安全帽检测怎么理解这些指标

假设验证集真实有：

- 100 个 `helmet`
- 40 个 `no-helmet`

模型输出里：

- 报了很多 `helmet`
- 其中一部分其实是背景误检

这会导致：

- `helmet` 的 `Precision` 下降

如果模型又漏掉了很多远处没戴帽的人：

- `no-helmet` 的 `Recall` 会低

如果框都能大致框住，但位置总差一点：

- `IoU` 可能偏低
- `mAP50-95` 也容易吃亏

所以指标分析一定要回到任务语境里看。

## 16. 初学者最容易混淆的 7 件事

### 16.1 Precision 不是“整体准确率”

它只关心：

**你报出来的目标里有多少是真的。**

### 16.2 Recall 不是“报得准不准”

它关心的是：

**你有没有把真实目标尽量都抓出来。**

### 16.3 IoU 不是类别指标

它主要看框位置重叠程度。

### 16.4 mAP 不是单一阈值下的普通平均

它背后依赖 AP 和 PR 曲线。

### 16.5 mAP50 高，不代表 mAP50-95 也高

说明模型可能“找得到”，但“框得不够精”。

### 16.6 loss 好看，不等于检测效果一定好

最终还是要看验证指标。

### 16.7 单看总 mAP 不够

还要看每个类别表现和坏样例。

## 复习速答

- `IoU`：框和框重叠程度。
- `Precision`：报出来的有多少是真的。
- `Recall`：真实目标里找到了多少。
- `mAP`：多个类别 AP 的平均。
- `mAP50-95`：比 `mAP50` 更严格。

## 19. 学完这一节后，你应该能回答的问题

1. IoU 到底在衡量什么
   - 预测框和真实框的**位置重合程度**。IoU = 交集面积 / 并集面积，0~1，越大越准。IoU≥0.5 表示"框的位置基本对了"。只衡量定位，不衡量分类对不对。

2. Precision 和 Recall 的区别是什么
   - Precision(精确率): 模型说"这是人"的所有框里，真正是人的比例。衡量"说的对不对"。高 Precision = 不乱报。
   - Recall(召回率): 所有真正的人里，被模型找到的比例。衡量"找的全不全"。高 Recall = 不漏人。
   - 区别: Precision 关注"预测质量"，Recall 关注"覆盖范围"。

3. 为什么 Precision 和 Recall 常常互相拉扯
   - conf 阈值调低 → 模型报更多框 → 覆盖更多真目标 → Recall↑，但也多了误检 → Precision↓
   - conf 阈值调高 → 只报高确定性的框 → 误检少 → Precision↑，但漏掉低分真目标 → Recall↓
   - 一个调高、另一个就降，是目标检测的永恒矛盾。F1 找平衡点。

4. F1 为什么是平衡指标
   - F1 = 2 × (P×R) / (P+R)。Precision 和 Recall 任何一个很低，F1 都会很低。只有两者都还行时 F1 才高。所以 F1 不偏向 Precision 或 Recall，是衡量"两边都还行"的综合分。

5. AP 和 mAP 的关系是什么
   - AP(每类): 对一个类，改 conf 阈值从 0~1，在每个点算 (Precision, Recall)，画 P-R 曲线，曲线下面积 = AP。
   - mAP(所有类): 所有类 AP 的平均值。YOLO 训练日志里的 mAP50 和 mAP50-95 就是这种。

6. `mAP50` 和 `mAP50-95` 为什么差很多
   - mAP50: 只要求 IoU≥0.5（框和真值重叠一半就算对）。宽松 → 分数虚高。0.8 很常见。
   - mAP50-95: IoU 从 0.5 到 0.95，步长 0.05（10 个点取平均）。0.95 要求框几乎完全贴合 → 严苛。通常 0.3~0.5。
   - 差很多 = 框的定位还不够精准，只是"大致在附近"。好模型的 mAP50-95 和 mAP50 差距更小。

7. 看到低 Precision、低 Recall、低 IoU 时，分别先怀疑什么
   - 低 Precision(误检多): conf 太低/NMS 不够狠/背景杂波多 → 先调高 conf 阈值或加负样本。
   - 低 Recall(漏检多): conf 太高把小目标过滤了/模型没学到这个类/该类数据太少 → 加该类数据或降 conf。
   - 低 mAP50-95(框不准): IoU 不高 → 标注质量差(框画得不准)/模型太小/特征提取不够强 → 检查标注 + 加深模型。
