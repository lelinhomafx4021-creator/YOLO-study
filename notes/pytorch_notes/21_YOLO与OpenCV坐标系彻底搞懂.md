## 💡 一、 计算机图像的坐标系长什么样？

**和数学课上的坐标系完全不一样！**

* 数学课：原点在左下角，Y 轴**向上**为正。
* 计算机图像：原点在**左上角**，Y 轴**向下**为正。

```text
(0,0) ────────────────────► X 轴 (向右为正，代表"宽度 Width")
  │
  │
  │      ┌──────────┐
  │      │  检测框    │
  │      │          │
  │      └──────────┘
  │
  ▼
Y 轴 (向下为正，代表"高度 Height")
```

> **大白话记忆法**：
> 想象你在读一篇文章。你的眼睛从**左上角**开始读，往右是 X，往下换行是 Y。
> 所以图片的第一个像素 `(0, 0)` 就在左上角。

## 🟩 二、 OpenCV 用的坐标格式：`xyxy`（左上角 + 右下角）

OpenCV 画矩形框 `cv2.rectangle()` 需要两个点：
* **左上角** `(x1, y1)` —— 框的起点
* **右下角** `(x2, y2)` —— 框的终点

```text
(0,0)
  ┌─────────────────────────────────────┐
  │                                     │
  │        (x1, y1)                     │
  │           ┌──────────────┐          │
  │           │              │          │
  │           │   检测到的    │          │
  │           │   安全帽      │          │
  │           │              │          │
  │           └──────────────┘          │
  │                       (x2, y2)      │
  │                                     │
  └─────────────────────────────────────┘
```

OpenCV 画框的代码：
```python
# 从左上角 (x1,y1) 画到右下角 (x2,y2)
cv2.rectangle(img, (x1, y1), (x2, y2), color=(0, 255, 0), thickness=2)
```

**YOLO 的 `box.xyxy` 输出的也是这个格式**：
```python
x1, y1, x2, y2 = box.xyxy[0].tolist()
# x1, y1 = 左上角坐标
# x2, y2 = 右下角坐标
```

所以当我们用 YOLO 的高层 API（`from ultralytics import YOLO`）时，拿到 `xyxy` 坐标可以直接丢给 `cv2.rectangle` 画框，非常方便。

## 🔵 三、 YOLO ONNX 输出的坐标格式：`xywh`（中心点 + 宽高）

**当我们脱离了 YOLO 框架，直接用 ONNX Runtime 跑底层推理时，模型输出的原始坐标格式完全不一样！**

ONNX 输出的每个检测框是 4 个数字：
* `cx` —— 检测框**中心点**的 X 坐标
* `cy` —— 检测框**中心点**的 Y 坐标
* `w`  —— 检测框的**宽度**
* `h`  —— 检测框的**高度**

```text
(0,0)
  ┌─────────────────────────────────────┐
  │                                     │
  │           ┌──────────────┐          │
  │           │              │          │
  │           │     (cx,cy)  │ ← 中心点 │
  │           │       ●      │          │
  │           │              │          │
  │           └──────────────┘          │
  │           |←───── w ────→|          │
  │                                     │
  └─────────────────────────────────────┘
```

## 🔄 四、 两种坐标格式之间的转换公式（面试必背）

### 从 `xywh`（中心点格式）转换为 `xyxy`（左上右下格式）：

```text
x1 = cx - w / 2    （中心点往左移半个宽度 = 左边界）
y1 = cy - h / 2    （中心点往上移半个高度 = 上边界）
x2 = cx + w / 2    （中心点往右移半个宽度 = 右边界）
y2 = cy + h / 2    （中心点往下移半个高度 = 下边界）
```

对应 Python 代码（就是我们 02_onnx_inference.py 里写的那段）：
```python
cx, cy, w, h = row[0:4]
x1 = int(cx - w / 2)
y1 = int(cy - h / 2)
x2 = int(cx + w / 2)
y2 = int(cy + h / 2)
```

### 从 `xyxy`（左上右下格式）转换为 `xywh`（中心点格式）：

```text
cx = (x1 + x2) / 2
cy = (y1 + y2) / 2
w  = x2 - x1
h  = y2 - y1
```

## ⚠️ 五、 NumPy 切片的坐标顺序（最容易搞混的坑！）

当我们用 NumPy 切片来"抠图"时，**Y 在前，X 在后**！这和上面所有的 `(x, y)` 写法正好是反过来的！

```python
# 抠出检测框区域的图像
crop = img[y1:y2, x1:x2]
#          ↑ Y 在前  ↑ X 在后
```

**为什么反过来？**
因为 OpenCV 读进来的图片在内存里是一个三维矩阵 `(Height, Width, Channels)`：
* 第一个维度是 Height（高度），对应的是 Y 轴
* 第二个维度是 Width（宽度），对应的是 X 轴

所以切片时必须 `[y:y, x:x]`，千万不能写成 `[x:x, y:y]`，否则抠出来的图是歪的或者直接报错。

## 📝 六、 总结速查表

| 场景 | 坐标格式 | 顺序 | 备注 |
| :--- | :--- | :--- | :--- |
| YOLO 高层 API `box.xyxy` | 左上角 + 右下角 | `x1, y1, x2, y2` | 可以直接给 OpenCV 画框 |
| ONNX 原始输出 | 中心点 + 宽高 | `cx, cy, w, h` | 需要手动转换为 xyxy 才能画框 |
| OpenCV `cv2.rectangle()` | 左上角 + 右下角 | `(x1,y1), (x2,y2)` | 注意是两个元组 |
| OpenCV `cv2.dnn.NMSBoxes()` | 左上角 + 宽高 | `x1, y1, w, h` | 注意不是 xyxy！ |
| NumPy 切片抠图 | Y 在前 X 在后 | `img[y1:y2, x1:x2]` | 和其他所有地方的顺序相反 |
| YOLO 标注文件 (.txt) | 归一化中心点+宽高 | `class cx cy w h` | 所有值都是 0~1 之间的比例值 |

## 🧪 七、亲手验证：坐标转换全流程

```python
import cv2

# 模拟一张 1920x1080 的工地监控图
W, H = 1920, 1080

# ── 标注文件里读到的（0~1 归一化值）──
# 格式: class cx cy w h
label = "1 0.532 0.418 0.087 0.095"  # no-helmet
cls_id, cx_n, cy_n, w_n, h_n = map(float, label.split())

# 1. 归一化 → 真实像素坐标（xywh 中心点格式）
cx = int(cx_n * W)    # 0.532 × 1920 = 1021
cy = int(cy_n * H)    # 0.418 × 1080 = 451
bw = int(w_n * W)     # 0.087 × 1920 = 167
bh = int(h_n * H)     # 0.095 × 1080 = 103

# 2. xywh → xyxy（中心点 → 左上/右下，方便 OpenCV 画框）
x1 = cx - bw // 2     # 1021 - 83 = 938
y1 = cy - bh // 2     # 451 - 51 = 400
x2 = cx + bw // 2     # 1021 + 83 = 1104
y2 = cy + bh // 2     # 451 + 51 = 502

# 3. OpenCV 画框（xyxy 格式）
frame = cv2.imread("test.jpg")
cv2.rectangle(frame, (x1, y1), (x2, y2), (0, 0, 255), 2)  # 红色框

# 4. NumPy 抠图（注意：先 Y 后 X！）
crop = frame[y1:y2, x1:x2]   # ✅ H在前, W在后
# crop = frame[x1:x2, y1:y2] # ❌ 这样会报错/出来歪的图
cv2.imwrite("crop.jpg", crop)
```

### 三种格式转换速算

```python
# xywh → xyxy
x1 = cx - w//2; y1 = cy - h//2; x2 = cx + w//2; y2 = cy + h//2

# xyxy → xywh
cx = (x1+x2)//2; cy = (y1+y2)//2; w = x2-x1; h = y2-y1

# 归一化(0~1) → 像素值
像素x = 归一化cx × 图片宽度; 像素w = 归一化w × 图片宽度
像素y = 归一化cy × 图片高度; 像素h = 归一化h × 图片高度
```

---

## 复习速答

- `xyxy`：左上角 + 右下角。
- `xywh`：中心点 + 宽高。
- `NMSBoxes`：要的是 `x, y, w, h`。
- `NumPy 切片`：要注意先 y 后 x。
- `YOLO 标注`：是归一化的中心点宽高。
