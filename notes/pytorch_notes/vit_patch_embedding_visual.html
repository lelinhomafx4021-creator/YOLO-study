ViT Patch Embedding 可视化

图片怎么切成 patch，再变成 Transformer 能吃的"词"序列

原始图片

用一张 8×8 的简化图（实际 ViT 用 224×224）。每个格子是一个像素，颜色代表 RGB 值。

鼠标悬停看像素值。这是模型的原始输入。

Step 1: 切成 Patch

切块

把 8×8 的图切成 4×4 的 patch（每个 patch 是 2×2 像素）。
8÷2 = 4 → 横向 4 块，纵向 4 块 → 共 16 个 patch。

Step 2: 展平成向量

展平

每个 2×2 的 patch 展平成一个 4 维向量（实际是 16×16×3 = 768 维）。
Patch 的像素排列成一行 → 就是一个向量。

Step 3: 加上 CLS Token + 位置编码

组装

CLS Token: 最前面加一个可学习的特殊向量（紫色），不对应任何 patch。经过 Transformer 后，它的输出就是分类结果。
位置编码: 每个向量加上位置信号（橙色虚线），告诉 Transformer 每个 patch 在图里的位置。

输入 = [CLS] + Patch_0 + Patch_1 + ... + Patch_15 + 位置编码
共 17 个向量（1 CLS + 16 patch），每个 4 维（实际 768 维）

Step 4: 送入 Transformer Encoder

17 个向量 → Transformer Encoder × 12 层 → 取 CLS Token 的输出 → Linear → 类别概率。

[CLS] [P0] [P1] ... [P15]
↓ Transformer Encoder × 12 ↓
[CLS输出] → Linear → 类别

完整流程

图片 224×224 → 切成 16×16 patch → 196 个 patch
→ 展平成 768 维向量 → 加 CLS Token + 位置编码 → 197 个向量
→ Transformer Encoder × 12 → 取 CLS 输出 → 分类
