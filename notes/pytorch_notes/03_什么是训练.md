# 什么是训练

## 这一节在整个学习顺序里的位置

如果我们按目前这条线学习：

1. `PyTorch 是什么`
2. `Tensor 是什么`
3. `什么是训练`
4. `什么是模型`
5. `完整训练流程`

那么学完 `Tensor` 之后，下一步最应该学的就是：

**训练到底在做什么。**

因为只有先搞懂“训练”，你后面学：

- `loss`
- `grad`
- `optimizer`
- `epoch`
- `batch`
- `model`

这些概念才不会变成一堆散的词。

所以这一节不是补充内容，而是主干。

---

## 先说最核心的一句话

**训练 = 让模型通过不断看数据、不断犯错、不断改参数，最后预测越来越准。**

也可以再压缩成一句：

**训练的本质，就是不断调整参数，让 `loss` 下降。**

这里有 3 个最关键的词：

- `参数 parameter`
- `损失 loss`
- `更新 update`

如果你只记住这一句，已经抓到了训练的核心。

---

## 1. 训练不是“运行模型”，而是“修改模型”

很多初学者会把“模型跑了一次”和“训练了一次”混为一谈。

这两个不是一回事。

### 1.1 只是运行模型

比如：

```python
y = model(x)
```

这只是：

- 把输入 `x` 喂给模型
- 得到一个预测结果 `y`

这一步叫：

**前向传播 forward**

它只是“算答案”，并没有让模型变聪明。

### 1.2 真正的训练

真正的训练还要继续往后做：

1. 得到预测
2. 和真实答案比较
3. 算出误差
4. 算出梯度
5. 更新参数

只有当参数真的被更新了，模型才算“学到了一点”。

所以：

- `model(x)` 不是完整训练
- `loss.backward()` 也不是完整训练
- `optimizer.step()` 之后，参数真的改了，这才叫训练往前走了一步

---

## 2. 什么叫“参数”

训练要改的，到底是什么？

答案是：

**模型内部的那些数字。**

这些数字就叫参数。

最简单的例子：

```python
y = w * x + b
```

这里：

- `x` 是输入
- `y` 是输出
- `w` 和 `b` 是参数

训练的目标，不是改 `x`，也不是改真实答案，而是改：

- `w`
- `b`

因为它们决定模型怎么做预测。

### 2.1 参数为什么重要

如果 `w` 和 `b` 很离谱，模型就会乱猜：

```python
w = 0.5
b = 10
x = 100

y = 0.5 * 100 + 10 = 60
```

但真实值可能是：

```python
target = 200
```

说明参数选得很差。

训练的过程，就是不断把这组参数从“乱猜”调整到“比较合理”。

---

## 3. 什么是 loss

训练里第二个核心概念是：

**loss = 当前模型错了多少。**

比如：

```python
pred = 60
target = 200
```

那说明模型猜错很多。

如果：

```python
pred = 198
target = 200
```

那就说明模型已经很接近了。

所以 `loss` 的作用非常明确：

**把“猜得准不准”变成一个可以计算的数。**

### 3.1 为什么一定要有 loss

因为训练不能靠“感觉差不多”。

我们需要一个明确的数字来判断：

- 现在比刚才更好了没有
- 这次参数更新是不是有效
- 模型有没有在进步

所以你可以把 `loss` 理解成：

**训练过程里的评分器。**

---

## 4. 什么是梯度

有了 `loss` 之后，还差最后一个问题：

**参数到底该怎么改？**

这就轮到梯度出场。

梯度的作用是：

**告诉你参数往哪个方向改，才能让 loss 变小。**

例如：

```python
w.grad
```

它本质上表示：

```python
d(loss) / d(w)
```

也就是：

**如果 `w` 变化一点点，`loss` 会怎么变化。**

### 4.1 怎么看梯度的正负

先记结论：

- `w.grad > 0`：`w` 应该变小
- `w.grad < 0`：`w` 应该变大

不是你手动规定它这样，而是参数更新公式决定的。

参数更新公式通常写成：

```python
w = w - lr * w.grad
```

所以：

- 如果 `w.grad` 是正数，减掉它，`w` 会变小
- 如果 `w.grad` 是负数，减掉它，`w` 会变大

### 4.2 梯度不是让你“手调”的

这一点非常重要。

初学时最容易误会成：

- `loss` 大了，要不要调梯度？

其实不是。

训练时：

- 梯度是 `backward()` 自动算出来的
- 你通常不手动去改梯度
- 你真正常调的是 `learning rate`

所以关系是：

- `loss`：错多少
- `grad`：该往哪边改
- `lr`：每次改多猛

---

## 5. 什么是学习率 learning rate

学习率的作用是：

**决定参数每次更新的幅度。**

公式还是这句：

```python
w = w - lr * w.grad
```

这里：

- `w.grad` 决定方向
- `lr` 决定步长

### 5.1 为什么需要学习率

因为即使方向是对的，也可能走太猛或者太慢。

#### 情况 1：学习率太大

你本来应该一点一点往谷底走，结果一步迈太大，直接跨过去了。

表现通常是：

- `loss` 剧烈震荡
- `loss` 不降反升
- 甚至训练发散

#### 情况 2：学习率太小

方向虽然对，但每次只挪一点点。

表现通常是：

- `loss` 下降得特别慢
- 训练半天没明显进展

### 5.2 一句话记忆

**梯度负责指路，学习率负责决定走多远。**

---

## 6. 什么是 optimizer

你已经知道了：

- 我们要用梯度改参数
- 改参数要乘一个学习率

那谁来做这件事？

答案是：

**优化器 optimizer**

优化器本质上就是：

**一个专门负责按照梯度更新参数的工具。**

最常见的写法：

```python
optimizer = torch.optim.SGD(model.parameters(), lr=0.01)
```

或者：

```python
optimizer = torch.optim.Adam(model.parameters(), lr=0.001)
```

### 6.1 optimizer 在训练里干嘛

它最直接的工作就是：

```python
optimizer.step()
```

这一步的含义是：

**根据当前已经算好的梯度，去更新模型参数。**

所以你可以这样记：

- `loss.backward()`：算出梯度
- `optimizer.step()`：真正改参数

这两个必须分开理解。

---

## 7. 什么是 epoch、batch、iteration

这三个词会一直出现，最好一开始就理顺。

### 7.1 epoch

`epoch` 的意思是：

**整个训练集完整看一遍。**

比如你有 100 张图片：

- 全部看完一次 = 1 个 epoch
- 全部看完 10 次 = 10 个 epoch

### 7.2 batch

`batch` 的意思是：

**一次送进模型的一小批数据。**

比如 `batch_size = 16`，表示每次喂 16 张图。

为什么不一次喂全部？

- 显存/内存不一定够
- 一次全喂计算代价很大

为什么不一次只喂 1 张？

- 太慢
- 梯度不稳定

所以训练通常是“按批次”进行。

### 7.3 iteration

每处理一个 batch，完成一次：

- 前向传播
- 算 loss
- 反向传播
- 更新参数

这通常就叫一次 `iteration` 或一步 `step`。

### 7.4 三者关系

举例：

- 训练集有 `100` 张图
- `batch_size = 20`

那么：

- 一个 epoch 里有 `5` 个 batch
- 也就是 `5` 次 iteration

如果训练 `10` 个 epoch`：

- 一共会更新参数 `50` 次

### 7.5 一句话关系图

```text
epoch
  包含多个 batch
  每个 batch 对应一次 iteration
```

---

## 8. 训练完整主线到底长什么样

现在把所有关键词串起来。

一次标准训练，大致是这个过程：

1. 取一批数据
2. 模型做预测
3. 计算 loss
4. 反向传播计算梯度
5. 优化器更新参数
6. 清空梯度
7. 继续下一批

写成流程图就是：

```text
输入 batch 数据
-> model 预测
-> 计算 loss
-> loss.backward() 计算梯度
-> optimizer.step() 更新参数
-> optimizer.zero_grad() 清空梯度
-> 下一个 batch
```

这条线，就是训练的骨架。

---

## 9. 最小训练例子：只有一个参数的世界

我们先不用完整神经网络，先看一个最小版本。

```python
import torch

w = torch.tensor(2.0, requires_grad=True)
x = torch.tensor(3.0)
target = torch.tensor(10.0)
lr = 0.1

y = w * x
loss = (y - target) ** 2

loss.backward()

with torch.no_grad():
    w = w - lr * w.grad

print(loss)
print(w)
```

### 9.1 这段代码到底在干嘛

先拆开看：

```python
w = 2
x = 3
```

所以：

```python
y = w * x = 6
```

但真实目标是：

```python
target = 10
```

所以：

```python
loss = (6 - 10)^2 = 16
```

说明现在错得比较多。

然后：

```python
loss.backward()
```

PyTorch 会自动算出：

```python
w.grad = -24
```

这个负号表示：

**当前应该把 `w` 往更大的方向调。**

再执行：

```python
w = w - 0.1 * (-24)
```

就会变成：

```python
w = 4.4
```

这说明：

- `loss` 告诉你现在错了多少
- `grad` 告诉你往哪改
- `lr` 决定改多猛

---

## 10. 为什么训练一定是“重复很多次”

很多人第一次看到训练代码时会问：

“为什么不算一次就完了？”

因为参数一开始通常是随机的，离最优解很远。

一次更新，只能让参数朝更好的方向动一点点。

所以训练本质上是：

**小步快跑，反复修正。**

这也是为什么你会看到：

```python
for epoch in range(...)
```

训练一定是循环，不是一次性完成。

---

## 11. PyTorch 里的标准训练代码长什么样

现在看一段最经典的 PyTorch 训练框架：

```python
for epoch in range(num_epochs):
    for x_batch, y_batch in dataloader:
        pred = model(x_batch)
        loss = criterion(pred, y_batch)

        optimizer.zero_grad()
        loss.backward()
        optimizer.step()
```

这一小段代码几乎是所有深度学习训练的母版。

### 11.1 逐行解释

```python
for epoch in range(num_epochs):
```

表示整个训练集要完整看很多遍。

```python
for x_batch, y_batch in dataloader:
```

表示每次只拿一个 batch 来训练。

```python
pred = model(x_batch)
```

前向传播：模型给出预测。

```python
loss = criterion(pred, y_batch)
```

把预测和真实答案做比较，得到损失。

```python
optimizer.zero_grad()
```

清空旧梯度。

这一句非常重要，因为 PyTorch 默认会把梯度累加。

```python
loss.backward()
```

反向传播：自动计算所有参数的梯度。

```python
optimizer.step()
```

根据梯度真正更新参数。

---

## 12. 为什么要先 `zero_grad()` 再 `backward()`

这是训练里最容易被机械记忆、但没真正理解的点。

PyTorch 默认的梯度机制是：

**累加梯度，而不是自动覆盖。**

也就是说，如果你不清空：

- 上一次 batch 的梯度还在
- 这一次的梯度又加上去

那参数更新就会混乱。

所以标准写法通常是：

```python
optimizer.zero_grad()
loss.backward()
optimizer.step()
```

你现在可以直接先把它记成固定模板。

---

## 13. 训练和推理的区别

这个也非常关键。

### 13.1 训练 training

训练时我们需要：

- 计算 loss
- 计算梯度
- 更新参数

所以训练阶段会有：

- `backward()`
- `optimizer.step()`

### 13.2 推理 inference

推理时我们只想：

- 输入一张图
- 让模型给结果

这时不需要：

- 算梯度
- 更新参数

所以推理和训练不是一回事。

你可以这样记：

- 训练：为了学习
- 推理：为了使用

---

## 14. 训练到底在优化什么

训练并不是在“直接优化准确率”。

训练时，真正直接优化的是：

**loss**

因为：

- 准确率通常不可导
- 而 loss 是专门设计出来可导、可优化的

所以训练更准确地说，是：

**通过梯度下降，不断让 loss 变小。**

当 loss 下降到一定程度，模型的准确率通常也会变好。

但这两者不是完全一回事。

## 15. 和 YOLO 的关系

你现在在做的目标检测项目，看上去像这样：

```python
model.train(data="data.yaml", epochs=10)
```

好像很简单。

但这背后实际上还是同一套训练骨架：

1. 读取一批图片和标注
2. 模型前向传播
3. 算检测 loss
4. 反向传播
5. 优化器更新参数
6. 重复很多轮

也就是说：

**YOLO 没有脱离训练本质，只是把底层流程封装起来了。**

所以你现在学这节，不是在学“底层和项目无关的理论”，而是在学你项目真正发生了什么。

## 16. 初学者最容易混淆的 6 件事

### 16.1 `loss` 不是参数

`loss` 只是“当前错了多少”的评分。

真正被改的是参数。

### 16.2 `grad` 不是你手调的值

梯度通常是自动算出来的。

你一般不手动调梯度本身。

### 16.3 `lr` 不决定方向

学习率只决定“改多猛”，不决定往哪改。

方向由梯度决定。

### 16.4 `backward()` 不会直接更新参数

它只负责算梯度。

真正更新参数的是：

```python
optimizer.step()
```

### 16.5 一个 epoch 不是一次参数更新

一个 epoch 里通常包含很多 batch，也就有很多次更新。

### 16.6 训练不是为了“记住数据”

训练的目标是让模型学到规律，而不是死记训练集。

## 18. 一张训练全景图

最后把所有概念串成一张图：

```text
训练数据
-> 按 batch 送进模型
-> 模型根据当前参数做预测
-> 用 loss 衡量预测错了多少
-> backward() 计算每个参数的梯度
-> optimizer 根据梯度更新参数
-> 一个 batch 结束
-> 全部 batch 跑完，叫一个 epoch
-> 重复多个 epoch，模型逐渐学会任务
```

如果你把这张图看顺了，训练这件事就真的入门了。

## 19. 学完这一节后，你应该能回答的问题

1. 训练的本质是什么
   - 通过大量"猜→对答案→调参数"的循环，让模型的参数从随机噪音变成能完成任务的数值。数学上：不断调整参数使损失函数值最小化。

2. 参数到底是什么
   - 模型里可以学习的数字（Conv2d 的卷积核权重、Linear 的 W 和 b）。初始是随机值，训练后变成有意义的模式。参数是模型的"知识"。

3. loss 为什么必须存在
   - 训练需要一个数值来评价"当前参数好不好"。loss 就是这个裁判：输出和正确答案差多少。没有 loss → 不知道参数该往哪调 → 训练无法进行。

4. 梯度到底告诉我们什么
   - 梯度 = 每个参数对 loss 的偏导数 = "这个参数调大一点，loss 会变大还是变小?"
   - 梯度为正 → 增大参数会让 loss 增大 → 应该减小参数
   - 梯度为负 → 增大参数会让 loss 减小 → 应该增大参数
   - 梯度的大小告诉你"这个参数对结果影响多大"

5. `loss.backward()` 和 `optimizer.step()` 的区别是什么
   - backward(): 从 loss 往回，用链式法则算出每个参数的梯度，存在 param.grad 里。只计算，不改参数。
   - step(): 用优化器规则（如 SGD: w = w - lr×grad），根据梯度更新参数值。只改参数，不算梯度。
   - 必须先 backward 再 step，否则 param.grad 是空的或旧值。

6. 学习率控制什么
   - 每次参数更新的步长。lr 太大 → 一步跨过最优解 → loss 震荡不收敛。lr 太小 → 走太慢 → 训练几百年也到不了。

7. epoch、batch、iteration 的关系是什么
   - 训练集 50000 张，batch_size=64:
       1 epoch  = 遍历整个训练集一次
       1 batch  = 64 张图，一起前向+反向传播
       1 iteration = 处理一个 batch 的一次更新
       epoch 内的 iteration 数 = ceil(50000/64) = 782

8. 为什么 YOLO 这种高层框架本质上还是这条训练主线
   - YOLO 的 model.train() 内部: 一样的 DataLoader → 一样的 forward → 一样的 loss(YOLO用三项loss) → backward → step。只是 YOLO 把这些包在一个方法里，但核心 API 就是 PyTorch 的 backward/step。看训练日志里 train/val 指标的变化规律，和手写 CNN 一模一样。

**什么是模型。**

- loss 在评价模型

**这些参数到底属于谁？模型到底是什么东西？**

## 复习速答

- 训练本质：不断“预测 -> 对答案 -> 算 loss -> 反传 -> 更新参数”。
- `loss`：衡量预测和真实答案差多少。
- `backward()`：算梯度，不改参数。
- `step()`：按梯度改参数。
- `epoch/batch`：一轮全数据 / 一小批数据。

这就是下一节的重点。
